## 1. Purpose

Provide a single, stable interface through which any part of the pipeline can
retrieve thermospheric wind components (v_zonal, v_meridional) at any geodetic
location and altitude, without knowing or caring which model or dataset is
behind it.

This module defines:

1. An abstract base class `WindMap` with one required method: `sample()`.
2. Four concrete subclasses — T1 through T4 — matching the truth wind
   datasets defined in the STM and traceable to the two WindCube science goals.
3. NetCDF save/load so any map can be serialised, shared, and reproduced exactly.

**Why this modularity matters.** The geometry pipeline (NB02), the FPI forward
model (M04), the Monte Carlo validation, and the INT notebooks all need wind
values at arbitrary lat/lon points. The source of those winds — a test constant,
an empirical climatology, a storm simulation — changes depending on what is
being tested. Hiding that behind `wind_map.sample(lat, lon)` means every
downstream module stays unchanged as the backend is swapped.

**On-orbit note.** When processing real flight data, `WindMap` is NOT called.
The FPI measures `v_rel` directly from the fringe Doppler shift. `WindMap` is
used only in the synthetic forward path to inject truth wind data for
simulation, calibration verification, and end-to-end validation. The NB02
functions that compute `V_sc_LOS` and `v_earth_LOS` run identically on real
and synthetic data — `WindMap` is only involved when constructing the
synthetic `v_wind_LOS` term.

## 2. Truth wind dataset definitions

Four backends, implemented in strict order. T1 must pass all tests before T2
is attempted; T2 before T3; T3 before T4.

| ID | Name | Description | STM pass criterion | Science link |
|----|------|-------------|-------------------|--------------|
| T1 | Uniform constant | Fixed v_zonal and v_merid everywhere | Bias < 5 m/s; spatially uniform recovery | Pipeline sanity check |
| T2 | Analytic gradient | Wind varies analytically with lat/lon | Gradient spatially preserved; RMS < 10 m/s | LOS geometry validation |
| T3 | HWM14 quiet-time | Empirical climatology, quiet conditions | Precision < 32 m/s; DE3 tidal phase recovered | SG2 — tides from below |
| T4 | Storm-perturbed | HWM14 + DWM07 (Kp-driven) or TIEGCM | Precision < 34 m/s; storm peak resolved | SG1 — forcing from above |

**STM precision derivation.** The amplitude precision formula
`σ(a) = sqrt(2/N) × σ(m)` (see STM v1) gives:
- SG1: N = 6 samples per storm event, σ(a) = 20 m/s → σ(m) < 34 m/s
- SG2: N = 84 samples for tidal ensemble, σ(a) = 5 m/s → σ(m) < 32 m/s

Both are satisfied by the STM projected performance of 9.8 m/s (1σ).

## 3. Interface contract — WindMap abstract base class

This contract is the only thing NB02 imports from this module. NB02 never
imports a concrete subclass directly — only the abstract `WindMap` type.

```python
from abc import ABC, abstractmethod
import numpy as np

class WindMap(ABC):
    """
    Abstract base class for thermospheric truth wind maps.

    All concrete subclasses must implement sample(). Grid convention:
        v_zonal : eastward wind component, m/s  (positive = eastward)
        v_merid : northward wind component, m/s (positive = northward)
        lat_grid : shape (180,), cell centres -89.5 to +89.5 deg
        lon_grid : shape (360,), cell centres -179.5 to +179.5 deg
        grids    : shape (180, 360), indexing [lat, lon]

    Wind sign conventions follow S02 Section 6.6:
        Zonal   positive = eastward
        Meridional positive = northward
    """

    @abstractmethod
    def sample(self, lat_deg: float, lon_deg: float) -> tuple[float, float]:
        """
        Return wind components at the given geodetic location.

        Parameters
        ----------
        lat_deg : float
            Geodetic latitude, degrees, -90 to +90.
        lon_deg : float
            Geodetic longitude, degrees, -180 to +180.
            Implementations must handle longitude wrapping (e.g. 181° → -179°).

        Returns
        -------
        (v_zonal_ms, v_merid_ms) : tuple[float, float]
            v_zonal_ms — eastward wind speed, m/s
            v_merid_ms — northward wind speed, m/s

        Notes
        -----
        Implementations must handle:
        - Longitude wrapping at ±180°
        - Pole clamping: lat_deg clamped to [-90, +90] if out of range
        - Bilinear interpolation for all grid-based backends (T2–T4)
        """

    def sample_array(
        self,
        lat_deg: np.ndarray,
        lon_deg: np.ndarray,
    ) -> tuple[np.ndarray, np.ndarray]:
        """
        Vectorised wrapper — call sample() over arrays of lat/lon.

        Parameters
        ----------
        lat_deg : np.ndarray, shape (N,)
        lon_deg : np.ndarray, shape (N,)

        Returns
        -------
        (v_zonal_ms, v_merid_ms) : tuple of np.ndarray, each shape (N,), float64
        """
        lat_deg = np.asarray(lat_deg, dtype=float)
        lon_deg = np.asarray(lon_deg, dtype=float)
        assert lat_deg.shape == lon_deg.shape
        v_z = np.empty(lat_deg.shape, dtype=float)
        v_m = np.empty(lat_deg.shape, dtype=float)
        for i in range(lat_deg.size):
            v_z.flat[i], v_m.flat[i] = self.sample(lat_deg.flat[i], lon_deg.flat[i])
        return v_z, v_m

    def plot(
        self,
        title: str = "WindMap",
        alt_km: float = 250.0,
        subsample: int = 5,
    ) -> None:
        """
        Diagnostic plot: filled-colour zonal and meridional wind maps
        on a global Cartopy Plate Carrée projection, with wind vector
        quivers overplotted at every `subsample`-th grid point.

        Uses matplotlib; does not return a figure object (displays inline
        or saves if a non-interactive backend is active).

        Parameters
        ----------
        title      : str — figure suptitle
        alt_km     : float — altitude label for the title
        subsample  : int  — plot every Nth grid point for quivers
        """
```

## 4. GridWindMap base class (internal)

`GridWindMap` is a concrete partial implementation of `WindMap` that provides:
- A 180×360 lat/lon grid stored as numpy arrays
- Bilinear interpolation via `scipy.interpolate.RegularGridInterpolator`
- `to_netcdf()` / `from_netcdf()` serialisation

T2, T3, and T4 all subclass `GridWindMap`. T1 (`UniformWindMap`) does not
need a grid and subclasses `WindMap` directly.

```python
from scipy.interpolate import RegularGridInterpolator

class GridWindMap(WindMap):
    """
    WindMap backed by a 180×360 lat/lon grid with bilinear interpolation.

    Subclasses must populate self.v_zonal_grid and self.v_merid_grid
    (both shape (180, 360), float32) in their __init__ before calling
    super().__init__() — or call _build_interpolators() explicitly
    after populating the grids.

    Grid layout:
        lat_grid[i] = -89.5 + i   degrees, i in 0..179
        lon_grid[j] = -179.5 + j  degrees, j in 0..359
    """

    LAT_GRID = np.linspace(-89.5, 89.5, 180)   # cell centres, degrees
    LON_GRID = np.linspace(-179.5, 179.5, 360)  # cell centres, degrees

    def _build_interpolators(self) -> None:
        """Build scipy RegularGridInterpolator objects from populated grids."""
        self._interp_zonal = RegularGridInterpolator(
            (self.LAT_GRID, self.LON_GRID),
            self.v_zonal_grid,
            method='linear',
            bounds_error=False,
            fill_value=None,   # extrapolate at poles
        )
        self._interp_merid = RegularGridInterpolator(
            (self.LAT_GRID, self.LON_GRID),
            self.v_merid_grid,
            method='linear',
            bounds_error=False,
            fill_value=None,
        )

    def sample(self, lat_deg: float, lon_deg: float) -> tuple[float, float]:
        """Bilinear interpolation at (lat_deg, lon_deg)."""
        # Wrap longitude to [-180, 180)
        lon_deg = ((lon_deg + 180.0) % 360.0) - 180.0
        # Clamp latitude
        lat_deg = float(np.clip(lat_deg, -90.0, 90.0))
        pt = np.array([[lat_deg, lon_deg]])
        vz = float(self._interp_zonal(pt))
        vm = float(self._interp_merid(pt))
        return (vz, vm)

    def to_netcdf(self, filepath: str, metadata: dict | None = None) -> None:
        """
        Save this wind map to a NetCDF4 file.

        File schema
        -----------
        Dimensions:
            lat : 180   (cell centres: -89.5, ..., +89.5 degrees)
            lon : 360   (cell centres: -179.5, ..., +179.5 degrees)
        Variables:
            lat      (lat,)      float32  degrees_north
            lon      (lon,)      float32  degrees_east   range -180 to +180
            v_zonal  (lat, lon)  float32  m/s  positive = eastward
            v_merid  (lat, lon)  float32  m/s  positive = northward
        Global attributes (all required):
            wind_map_type    : str  'T1_uniform' | 'T2_analytic' | 'T3_hwm14' | 'T4_storm'
            created_utc      : str  ISO 8601 timestamp (UTC)
            alt_km           : float  nominal wind altitude in km
            description      : str  human-readable description
            pipeline_version : str  git commit hash or 'dev'

        Parameters
        ----------
        filepath : str — path to output .nc file
        metadata : dict | None — extra key-value pairs added as global attributes
        """

    @classmethod
    def from_netcdf(cls, filepath: str) -> 'GridWindMap':
        """
        Load a previously saved GridWindMap from a NetCDF4 file.

        Returns a GridWindMap instance with v_zonal_grid and v_merid_grid
        populated from the file and interpolators initialised.

        The returned instance has `wind_map_type` set from the file's
        global attribute of the same name.
        """
```

## 5. T1 — UniformWindMap

```python
class UniformWindMap(WindMap):
    """
    Uniform wind field: identical v_zonal and v_merid at every location.

    Purpose
    -------
    Simplest possible pipeline sanity check. If the end-to-end pipeline
    cannot recover a spatially uniform 100 m/s zonal wind exactly,
    something is fundamentally wrong in the geometry or inversion.

    No external dependencies. No grid required.

    Parameters
    ----------
    v_zonal_ms : float
        Eastward wind, m/s, everywhere on Earth. Default 100.0.
    v_merid_ms : float
        Northward wind, m/s, everywhere on Earth. Default 0.0.

    Usage
    -----
    wind_map = UniformWindMap(v_zonal_ms=100.0, v_merid_ms=50.0)
    vz, vm = wind_map.sample(lat_deg=30.0, lon_deg=45.0)
    # Returns (100.0, 50.0) for any lat/lon
    """

    def __init__(self, v_zonal_ms: float = 100.0, v_merid_ms: float = 0.0):
        self.v_zonal_ms = float(v_zonal_ms)
        self.v_merid_ms = float(v_merid_ms)

    def sample(self, lat_deg: float, lon_deg: float) -> tuple[float, float]:
        return (self.v_zonal_ms, self.v_merid_ms)
```

**T1 verification requirement:** `sample()` must return exactly
`(v_zonal_ms, v_merid_ms)` for any lat/lon, including poles (±90°),
the prime meridian (0°), and the antimeridian (±180°).

## 6. T2 — AnalyticWindMap

```python
class AnalyticWindMap(GridWindMap):
    """
    Analytic wind field with known spatial structure.

    Purpose
    -------
    Tests that tangent-point sampling correctly reproduces spatial variation.
    Uses closed-form formulas — results are exact and fully reproducible
    with no external dependencies.

    Two patterns, selected by the `pattern` argument:

    Pattern 'sine_lat' (default):
        v_zonal(lat) = A_z * sin(lat_rad)
        v_merid(lat) = A_m * cos(lat_rad)
        with A_z = 200.0 m/s, A_m = 100.0 m/s (default)

        Physical meaning: a latitude-symmetric zonal jet peaking at ±90°
        with a meridional cross-equatorial flow. Zonal is zero at equator,
        meridional is maximum at equator. This pattern tests that the
        pipeline correctly handles the equatorial-to-polar gradient that
        is relevant for the STM DE3 tidal science.

    Pattern 'wave4' (DE3-like):
        v_zonal(lat, lon) = A_z * sin(lat_rad) * cos(4 * lon_rad + phi)
        v_merid(lat, lon) = A_m * cos(lat_rad) * sin(4 * lon_rad + phi)
        with A_z = 150.0 m/s, A_m = 75.0 m/s, phi = 0.0 rad (default)

        Physical meaning: a wave-4 longitudinal modulation of the zonal
        wind, mimicking the DE3 (diurnal eastward wavenumber 3) non-migrating
        tide that is Science Goal 2's primary observable. This is the T2
        pattern used for SG2 tidal validation.

    Parameters
    ----------
    pattern    : str   'sine_lat' or 'wave4'. Default 'sine_lat'.
    A_zonal_ms : float Zonal amplitude, m/s.
    A_merid_ms : float Meridional amplitude, m/s.
    phase_rad  : float Phase offset for wave4 pattern, radians. Default 0.0.
    alt_km     : float Altitude label for NetCDF output. Default 250.0.
    """

    def __init__(
        self,
        pattern: str = 'sine_lat',
        A_zonal_ms: float = 200.0,
        A_merid_ms: float = 100.0,
        phase_rad: float = 0.0,
        alt_km: float = 250.0,
    ):
```

**Grid population formulas (implement these exactly):**

```python
# For 'sine_lat':
lat_rad = np.radians(self.LAT_GRID)           # shape (180,)
v_zonal_grid[i, :] = A_zonal_ms * np.sin(lat_rad[i])   # same for all lon
v_merid_grid[i, :]  = A_merid_ms * np.cos(lat_rad[i])  # same for all lon

# For 'wave4':
lat_rad = np.radians(self.LAT_GRID)           # shape (180,)
lon_rad = np.radians(self.LON_GRID)           # shape (360,)
LAT, LON = np.meshgrid(lat_rad, lon_rad, indexing='ij')  # (180, 360)
v_zonal_grid = A_zonal_ms * np.sin(LAT) * np.cos(4 * LON + phase_rad)
v_merid_grid  = A_merid_ms * np.cos(LAT) * np.sin(4 * LON + phase_rad)
```

After populating, call `self._build_interpolators()`.

## 7. T3 — HWM14WindMap

```python
class HWM14WindMap(GridWindMap):
    """
    Empirical quiet-time thermospheric wind map from HWM14.

    Purpose
    -------
    Provides a physically realistic quiet-time wind field for pipeline
    validation under nominal solar and geomagnetic conditions.
    Linked to Science Goal 2 (tides from below): the HWM14 model captures
    the diurnal migrating tide and the DE3 non-migrating tide at 250 km.

    HWM14 reference: Drob et al. (2015), doi:10.1002/2014EA000089

    Parameters
    ----------
    alt_km       : float  Altitude for HWM14 query, km. Default 250.0
                          (from S03 TP_ALTITUDE_KM).
    day_of_year  : int    Day of year (1–365). Default 172 (June 21, summer
                          solstice — typical for operational planning).
    ut_hours     : float  Universal time in hours (0.0–24.0). Default 12.0.
    f107         : float  10.7 cm solar flux index. Default 150.0 (moderate
                          solar activity). Range 70–250.
    f107a        : float  81-day average F10.7. Default 150.0.
    ap           : float  Geomagnetic ap index. Default 4 (quiet conditions,
                          Kp ≈ 1). Set to 80 for Kp ≈ 6 storm conditions (T4).

    Dependencies
    ------------
    hwm14 Python package. Install:
        conda install -c conda-forge hwm14    (recommended)
        pip install hwm14                     (requires gfortran)

    If hwm14 is not installed, tests are skipped via pytest.importorskip().

    Usage
    -----
    wm = HWM14WindMap(alt_km=250.0, day_of_year=172, ut_hours=12.0,
                      f107=150.0, f107a=150.0, ap=4)
    vz, vm = wm.sample(lat_deg=30.0, lon_deg=120.0)
    """
```

**Implementation note:** Call `hwm14.hwm14()` at every grid point (180 × 360 = 64,800 calls)
during `__init__`. This takes ~10–60 seconds depending on hardware. After construction,
call `to_netcdf()` to save the result. On subsequent runs, load from NetCDF with
`GridWindMap.from_netcdf()` — this is instantaneous. The spec tests should construct
from NetCDF if the file exists, and from HWM14 only if it does not.

**HWM14 function signature:**
```python
import hwm14
# hwm14.hwm14(iyd, sec, alt, glat, glon, stl, f107a, f107, ap, ap_array)
# iyd  : integer year + day-of-year (e.g. 2027172 for day 172 of 2027)
# sec  : seconds since midnight UTC
# alt  : altitude in km
# glat : geodetic latitude, degrees
# glon : geographic longitude, degrees
# stl  : local solar time, hours (can pass -1 for automatic)
# ap   : array of length 7; set all to ap value for simplified use
result = hwm14.hwm14(iyd, sec, alt_km, lat_deg, lon_deg, -1, f107a, f107, 0,
                     [ap]*7)
v_merid = result[0]  # m/s, northward  (hwm14 convention: first = meridional)
v_zonal  = result[1]  # m/s, eastward
```

**Expected output at 250 km for quiet conditions (sanity check values):**

| Condition | v_zonal range | v_merid range |
|-----------|---------------|---------------|
| Quiet equatorial day | 0–200 m/s eastward | ±100 m/s |
| Quiet equatorial night | 50–300 m/s eastward | ±50 m/s |
| Quiet high latitude (±60°) | ±300 m/s | ±150 m/s |

These are approximate ranges from the HWM14-driven maps visible in the
WindCube Continuation Review deck (Deck 13). If v_zonal exceeds 500 m/s
or v_merid exceeds 400 m/s at any grid point, something is wrong with
the HWM14 call (likely a unit error — HWM14 outputs m/s, not km/s).

## 8. T4 — StormWindMap

```python
class StormWindMap(GridWindMap):
    """
    Storm-perturbed thermospheric wind map for geomagnetic storm simulation.

    Purpose
    -------
    Represents the wind state during a G2-class geomagnetic storm (Kp ≥ 6),
    which is Science Goal 1's primary observable. Peak equatorward surge
    winds can reach 400 m/s at mid-latitudes.

    Two backends are supported. Use Backend A unless TIEGCM data is available.

    Backend A — HWM14 + DWM07 (default, no extra data required):
        Set ap = 80 (corresponds to Kp ≈ 6, G2 storm).
        HWM14 internally calls DWM07 for the disturbance wind component.

    Backend B — TIEGCM NetCDF (highest fidelity):
        Load time-interpolated wind fields from a TIEGCM simulation NetCDF
        file. TIEGCM output is on pressure-level grids; interpolation to
        fixed altitude requires height-to-pressure mapping.

    For the purposes of S05, Backend A is the required implementation.
    Backend B is optional and may be deferred to a follow-up spec.

    Parameters
    ----------
    alt_km       : float   Altitude, km. Default from S03 TP_ALTITUDE_KM.
    day_of_year  : int     Day of year. Default 355 (December 21, eclipse season).
    ut_hours     : float   UT hours. Default 3.0 (post-midnight, peak storm phase).
    f107         : float   F10.7 index. Default 180.0 (elevated solar activity).
    f107a        : float   81-day average F10.7. Default 180.0.
    ap           : float   ap index. Default 80 (Kp ≈ 6, G2 storm).
                           Use 179 for Kp = 9 (extreme storm).

    Usage
    -----
    wm = StormWindMap(ap=80)   # Kp ≈ 6 storm
    vz, vm = wm.sample(lat_deg=45.0, lon_deg=0.0)
    # Expected: large equatorward meridional wind (negative v_merid in NH)
    """
```

**Expected characteristics for a G2 storm (sanity check):**
- `v_merid` at 45°N should be strongly negative (equatorward, −100 to −400 m/s)
- `v_zonal` at low latitudes should be eastward and enhanced vs. quiet-time
- The quiet-to-storm difference `(StormWindMap - HWM14WindMap)` should peak
  at auroral latitudes (~60–70°) and decay toward the equator

**TIEGCM reference data:** If TIEGCM NetCDF files are available, they can be
obtained from the CEDAR Madrigal database at
`https://cedar.openmadrigal.org` or the NCAR HPSS archive. The relevant
simulation is the standard TIEGCM2.0 geomagnetic storm run. Contact
the NCAR Whole Atmosphere Modeling group for pre-computed storm NetCDF files.

def test_T2_wave4_longitude_cycles():
    import numpy as np
    wm = AnalyticWindMap(pattern='wave4', A_zonal_ms=150.0, A_merid_ms=75.0)
    lons = np.linspace(-180, 180, 361)
    vz_vals = np.array([wm.sample(30.0, lon)[0] for lon in lons])
    # Count zero-crossings of vz at fixed lat=30 — expect 8 (4 full cycles = 8 crossings)
    crossings = np.sum(np.diff(np.sign(vz_vals)) != 0)

## 12. External validation reference

For visual and numerical validation of T3 and T4 output, compare against:

- **HWM14 paper:** Drob et al. (2015), *Earth and Space Science*, 2,
  doi:10.1002/2014EA000089. Figures 4–6 show zonal and meridional wind maps
  at 250 km for quiet conditions — the pattern should qualitatively match
  the T3 output.
- **ICON/MIGHTI L3 winds:** NASA Earthdata, ICON mission. At 250 km,
  quiet-time zonal winds of 100–300 m/s eastward at low-to-mid latitudes are
  typical. Download via `earthdata.nasa.gov` (search "ICON_L3-7_MIGHTI").
- **TIE-GCM:** For T4 storm validation, TIE-GCM output is publicly accessible
  via the CEDAR Madrigal database (`cedar.openmadrigal.org`). The relevant
  run is TIEGCM2.0 with Weimer electric field model and GSWM tidal forcing.
