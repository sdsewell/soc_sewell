## 1. Purpose

Provide a single, stable interface through which any part of the pipeline can
retrieve thermospheric wind components (v_zonal, v_meridional) at any geodetic
location and altitude, without knowing or caring which model or dataset is
behind it.

This module defines:

1. An abstract base class `WindMap` with one required method: `sample()`.
2. Four concrete subclasses — T1 through T4 — matching the truth wind
   datasets defined in the STM and traceable to the two WindCube science goals.
3. NetCDF save/load so any map can be serialised, shared, and reproduced exactly.
4. A four-mode `plot()` method for diagnostic and poster-quality visualisation
   of the full 2-D wind vector field.

**Why this modularity matters.** The geometry pipeline (NB02), the FPI forward
model (M04), the Monte Carlo validation, and the INT notebooks all need wind
values at arbitrary lat/lon points. The source of those winds — a test constant,
an empirical climatology, a storm simulation — changes depending on what is
being tested. Hiding that behind `wind_map.sample(lat, lon)` means every
downstream module stays unchanged as the backend is swapped.

**On-orbit note.** When processing real flight data, `WindMap` is NOT called.
The FPI measures `v_rel` directly from the fringe Doppler shift. `WindMap` is
used only in the synthetic forward path to inject truth wind data for
simulation, calibration verification, and end-to-end validation. The NB02
functions that compute `V_sc_LOS` and `v_earth_LOS` run identically on real
and synthetic data — `WindMap` is only involved when constructing the
synthetic `v_wind_LOS` term.

## 2. Truth wind dataset definitions

Four backends, implemented in strict order. T1 must pass all tests before T2
is attempted; T2 before T3; T3 before T4.

| ID | Name | Description | STM pass criterion | Science link |
|----|------|-------------|-------------------|--------------|
| T1 | Uniform constant | Fixed v_zonal and v_merid everywhere | Bias < 5 m/s; spatially uniform recovery | Pipeline sanity check |
| T2 | Analytic gradient | Wind varies analytically with lat/lon | Gradient spatially preserved; RMS < 10 m/s | LOS geometry validation |
| T3 | HWM14 quiet-time | Empirical climatology, quiet conditions | Precision < 32 m/s; DE3 tidal phase recovered | SG2 — tides from below |
| T4 | Storm-perturbed | HWM14 + DWM07 (Kp-driven) or TIEGCM | Precision < 34 m/s; storm peak resolved | SG1 — forcing from above |

**STM precision derivation.** The amplitude precision formula
`σ(a) = sqrt(2/N) × σ(m)` (see STM v1) gives:
- SG1: N = 6 samples per storm event, σ(a) = 20 m/s → σ(m) < 34 m/s
- SG2: N = 84 samples for tidal ensemble, σ(a) = 5 m/s → σ(m) < 32 m/s

Both are satisfied by the STM projected performance of 9.8 m/s (1σ).

## 3. Interface contract — WindMap abstract base class

This contract is the only thing NB02 imports from this module. NB02 never
imports a concrete subclass directly — only the abstract `WindMap` type.

```python
from abc import ABC, abstractmethod
import numpy as np

class WindMap(ABC):
    """
    Abstract base class for thermospheric truth wind maps.

    All concrete subclasses must implement sample(). Grid convention:
        v_zonal : eastward wind component, m/s  (positive = eastward)
        v_merid : northward wind component, m/s (positive = northward)
        lat_grid : shape (180,), cell centres -89.5 to +89.5 deg
        lon_grid : shape (360,), cell centres -179.5 to +179.5 deg
        grids    : shape (180, 360), indexing [lat, lon]

    Wind sign conventions follow S02 Section 6.6:
        Zonal      positive = eastward
        Meridional positive = northward
    """

    @abstractmethod
    def sample(self, lat_deg: float, lon_deg: float) -> tuple[float, float]:
        """
        Return wind components at the given geodetic location.

        Parameters
        ----------
        lat_deg : float
            Geodetic latitude, degrees, -90 to +90.
        lon_deg : float
            Geodetic longitude, degrees, -180 to +180.
            Implementations must handle longitude wrapping (e.g. 181° → -179°).

        Returns
        -------
        (v_zonal_ms, v_merid_ms) : tuple[float, float]
            v_zonal_ms — eastward wind speed, m/s
            v_merid_ms — northward wind speed, m/s

        Notes
        -----
        Implementations must handle:
        - Longitude wrapping at ±180°
        - Pole clamping: lat_deg clamped to [-90, +90] if out of range
        - Bilinear interpolation for all grid-based backends (T2–T4)
        """

    def sample_array(
        self,
        lat_deg: np.ndarray,
        lon_deg: np.ndarray,
    ) -> tuple[np.ndarray, np.ndarray]:
        """
        Vectorised wrapper — call sample() over arrays of lat/lon.

        Parameters
        ----------
        lat_deg : np.ndarray, shape (N,)
        lon_deg : np.ndarray, shape (N,)

        Returns
        -------
        (v_zonal_ms, v_merid_ms) : tuple of np.ndarray, each shape (N,), float64
        """
        lat_deg = np.asarray(lat_deg, dtype=float)
        lon_deg = np.asarray(lon_deg, dtype=float)
        assert lat_deg.shape == lon_deg.shape
        v_z = np.empty(lat_deg.shape, dtype=float)
        v_m = np.empty(lat_deg.shape, dtype=float)
        for i in range(lat_deg.size):
            v_z.flat[i], v_m.flat[i] = self.sample(lat_deg.flat[i], lon_deg.flat[i])
        return v_z, v_m

    # ------------------------------------------------------------------
    # Plotting interface
    # ------------------------------------------------------------------

    def plot(
        self,
        title: str = "WindMap",
        alt_km: float = 250.0,
        subsample: int = 5,
        mode: str = 'separate',
        save_path: str | None = None,
    ) -> None:
        """
        Visualise the wind field on a global Plate Carrée projection.

        Four modes are available, selected by the `mode` parameter:

        mode='separate'  (default — preserved from previous spec version)
            Two-panel figure. Left panel: v_zonal filled-colour map (RdBu_r,
            ±400 m/s). Right panel: v_merid filled-colour map (RdBu_r,
            ±200 m/s). Wind vector quivers overplotted at every
            `subsample`-th grid point on both panels.
            Best for: direct comparison of U and V components side-by-side;
            pipeline diagnostic output.

        mode='vector'
            Single-panel figure. Wind speed magnitude (sqrt(U²+V²)) shown as
            a filled-colour map (viridis, 0 to v_max). Wind direction shown as
            white quiver arrows overplotted at every `subsample`-th point.
            Colourbar labelled "Wind speed (m/s)".
            Best for: showing both magnitude and direction in one panel;
            science paper figures; quick inspection of storm perturbations.

        mode='stream'
            Single-panel figure. Streamlines computed from the full-resolution
            U and V grids using matplotlib streamplot, coloured by wind speed
            magnitude (plasma colourmap). Streamline density = 1.5, linewidth
            scaled by speed. No quiver arrows.
            Best for: visualising large-scale circulation patterns (vortices,
            convergence zones, jet streams); poster panels.

        mode='magnitude'
            Single-panel figure. Wind speed magnitude only, as a filled-colour
            map (hot_r, 0 to v_max). No arrows or streamlines. A thin black
            contour is drawn at the STM precision threshold (WIND_BIAS_BUDGET_MS
            from src.constants, nominally 9.8 m/s) to show where wind gradients
            exceed the measurement noise floor.
            Best for: identifying high-wind regions; STM traceability figures.

        Parameters
        ----------
        title      : str   — figure suptitle (default 'WindMap')
        alt_km     : float — altitude label appended to the suptitle (default 250.0)
        subsample  : int   — plot every Nth grid point for quivers; ignored for
                             'stream' and 'magnitude' modes (default 5)
        mode       : str   — one of 'separate', 'vector', 'stream', 'magnitude'
                             (default 'separate')
        save_path  : str | None — if given, save figure to this path instead of
                             displaying; format inferred from extension
                             (e.g. '.png', '.pdf'). Default None (display inline).

        Raises
        ------
        ValueError
            If `mode` is not one of the four recognised strings.

        Notes
        -----
        All four modes require matplotlib and cartopy. For `mode='stream'` the
        full 180×360 grid is passed to matplotlib.streamplot; no subsampling is
        applied to the underlying grid (streamplot handles its own density
        parameter). For `mode='separate'` and `mode='vector'`, quiver arrows
        are subsampled at every `subsample`-th point in both lat and lon.

        The `UniformWindMap` (T1) does not store a grid internally. For T1
        instances, `plot()` builds a temporary 180×360 evaluation grid by
        calling `sample_array()` before plotting. This is transparent to the
        caller.
        """
        _VALID_MODES = ('separate', 'vector', 'stream', 'magnitude')
        if mode not in _VALID_MODES:
            raise ValueError(
                f"plot() mode='{mode}' not recognised. "
                f"Valid options: {_VALID_MODES}"
            )
        # Build evaluation grid (works for all subclasses including T1)
        vz_grid, vm_grid = self._build_plot_grid()
        full_title = f"{title}  |  alt = {alt_km:.0f} km  |  mode = {mode}"

        if mode == 'separate':
            self._plot_separate(vz_grid, vm_grid, full_title, subsample, save_path)
        elif mode == 'vector':
            self._plot_vector(vz_grid, vm_grid, full_title, subsample, save_path)
        elif mode == 'stream':
            self._plot_stream(vz_grid, vm_grid, full_title, save_path)
        elif mode == 'magnitude':
            self._plot_magnitude(vz_grid, vm_grid, full_title, save_path)
```

### 3.1 `_build_plot_grid()` helper

```python
def _build_plot_grid(self) -> tuple[np.ndarray, np.ndarray]:
    """
    Return (vz_grid, vm_grid), each shape (180, 360), for plotting.

    For GridWindMap subclasses (T2–T4), returns self.v_zonal_grid and
    self.v_merid_grid directly (no recomputation).

    For UniformWindMap (T1), evaluates sample_array() over the standard
    LAT_GRID × LON_GRID mesh and returns the result.
    """
    if hasattr(self, 'v_zonal_grid'):
        return self.v_zonal_grid, self.v_merid_grid
    # T1 fallback: evaluate on standard grid
    from src.windmap.nb00_wind_map import GridWindMap   # local import avoids circularity
    lats = GridWindMap.LAT_GRID
    lons = GridWindMap.LON_GRID
    LAT, LON = np.meshgrid(lats, lons, indexing='ij')   # (180, 360)
    vz, vm = self.sample_array(LAT.ravel(), LON.ravel())
    return vz.reshape(180, 360), vm.reshape(180, 360)
```

### 3.2 `_plot_separate()` — two-panel U / V maps

```python
def _plot_separate(
    self,
    vz_grid: np.ndarray,
    vm_grid: np.ndarray,
    title: str,
    subsample: int,
    save_path: str | None,
) -> None:
    """
    Two-panel figure: v_zonal (left) and v_merid (right).

    Both panels use a diverging RdBu_r colourmap centred on zero.
    Quiver arrows are overplotted at every `subsample`-th grid point.

    Colour limits:
        v_zonal : vmin = -400 m/s, vmax = +400 m/s
        v_merid : vmin = -200 m/s, vmax = +200 m/s
    These are chosen to span the full HWM14 / storm range without
    saturating the quiet-time signal.

    Layout: 1 row × 2 columns, figure size (14, 5) inches.
    """
    import matplotlib.pyplot as plt
    import cartopy.crs as ccrs
    import cartopy.feature as cfeature

    from src.windmap.nb00_wind_map import GridWindMap
    lats = GridWindMap.LAT_GRID
    lons = GridWindMap.LON_GRID
    LON2D, LAT2D = np.meshgrid(lons, lats)          # shape (180, 360), broadcast only
    # Quiver subsampling indices
    si = slice(None, None, subsample)

    fig, axes = plt.subplots(
        1, 2, figsize=(14, 5),
        subplot_kw={'projection': ccrs.PlateCarree()},
    )
    fig.suptitle(title, fontsize=11)

    panels = [
        (axes[0], vz_grid, 'Zonal wind  U  (m/s)',      400, 'RdBu_r'),
        (axes[1], vm_grid, 'Meridional wind  V  (m/s)', 200, 'RdBu_r'),
    ]
    for ax, grid, label, vmax, cmap in panels:
        ax.add_feature(cfeature.COASTLINE, linewidth=0.5)
        ax.add_feature(cfeature.BORDERS,   linewidth=0.3)
        ax.gridlines(draw_labels=True, linewidth=0.3, alpha=0.5)
        im = ax.pcolormesh(lons, lats, grid,
                           vmin=-vmax, vmax=vmax,
                           cmap=cmap,
                           transform=ccrs.PlateCarree())
        plt.colorbar(im, ax=ax, orientation='horizontal',
                     pad=0.05, label=label)
        # Quiver arrows (subsampled)
        ax.quiver(
            LON2D[si, si], LAT2D[si, si],
            vz_grid[si, si], vm_grid[si, si],
            transform=ccrs.PlateCarree(),
            color='black', alpha=0.6,
            width=0.003, scale=4000,
        )

    plt.tight_layout()
    if save_path:
        plt.savefig(save_path, dpi=150, bbox_inches='tight')
        plt.close(fig)
    else:
        plt.show()
```

### 3.3 `_plot_vector()` — speed colour + quiver direction

```python
def _plot_vector(
    self,
    vz_grid: np.ndarray,
    vm_grid: np.ndarray,
    title: str,
    subsample: int,
    save_path: str | None,
) -> None:
    """
    Single-panel figure: wind speed magnitude as colour, direction as arrows.

    Colourmap: 'viridis', range [0, v_max] where v_max = 98th percentile
    of the speed distribution (prevents a handful of extreme storm values
    from compressing the colour scale for the bulk of the field).

    Quiver arrows: white, semi-transparent (alpha=0.75), overplotted at
    every `subsample`-th grid point. Arrow length is normalised (scale=1,
    units='xy') so all arrows are the same length — direction only, not
    magnitude (magnitude is already conveyed by colour).

    Colourbar: horizontal, below the axes, labelled 'Wind speed (m/s)'.

    Layout: single axes, figure size (10, 5) inches.

    Physical interpretation note: this mode is the most information-dense
    single-panel wind plot. Colour immediately shows the high-speed regions
    (storm surges, thermospheric jets); arrows show whether the flow is
    zonal, meridional, or oblique. For storm cases (T4), the combination of
    high speed and equatorward arrows at mid-latitudes is immediately
    recognisable as a geomagnetic storm signature.
    """
    import matplotlib.pyplot as plt
    import cartopy.crs as ccrs
    import cartopy.feature as cfeature

    from src.windmap.nb00_wind_map import GridWindMap
    lats = GridWindMap.LAT_GRID
    lons = GridWindMap.LON_GRID
    LON2D, LAT2D = np.meshgrid(lons, lats)

    speed = np.sqrt(vz_grid**2 + vm_grid**2)
    v_max = float(np.nanpercentile(speed, 98))
    # Normalised unit vectors for direction-only quivers
    with np.errstate(invalid='ignore'):
        u_norm = np.where(speed > 0, vz_grid / speed, 0.0)
        v_norm = np.where(speed > 0, vm_grid / speed, 0.0)

    si = slice(None, None, subsample)

    fig, ax = plt.subplots(
        1, 1, figsize=(10, 5),
        subplot_kw={'projection': ccrs.PlateCarree()},
    )
    fig.suptitle(title, fontsize=11)
    ax.add_feature(cfeature.COASTLINE, linewidth=0.5)
    ax.add_feature(cfeature.BORDERS,   linewidth=0.3)
    ax.gridlines(draw_labels=True, linewidth=0.3, alpha=0.5)

    im = ax.pcolormesh(lons, lats, speed,
                       vmin=0, vmax=v_max,
                       cmap='viridis',
                       transform=ccrs.PlateCarree())
    plt.colorbar(im, ax=ax, orientation='horizontal',
                 pad=0.05, label='Wind speed (m/s)')

    # Direction-only arrows (normalised, same length everywhere)
    ax.quiver(
        LON2D[si, si], LAT2D[si, si],
        u_norm[si, si], v_norm[si, si],
        transform=ccrs.PlateCarree(),
        color='white', alpha=0.75,
        width=0.003, scale=30, headwidth=4,
    )

    plt.tight_layout()
    if save_path:
        plt.savefig(save_path, dpi=150, bbox_inches='tight')
        plt.close(fig)
    else:
        plt.show()
```

### 3.4 `_plot_stream()` — streamlines coloured by speed

```python
def _plot_stream(
    self,
    vz_grid: np.ndarray,
    vm_grid: np.ndarray,
    title: str,
    save_path: str | None,
) -> None:
    """
    Single-panel figure: streamlines of the vector wind field.

    Uses matplotlib.pyplot.streamplot on the full 180×360 grid.
    Streamlines are coloured by local wind speed (plasma colourmap).
    Line width is also proportional to speed (linewidth range 0.5–2.5).

    streamplot parameters:
        density   : 1.5  (controls how many streamlines are drawn)
        arrowsize : 1.2
        color     : speed array (plasma colourmap, vmin=0, vmax=98th pct)
        linewidth : speed / v_scale  (v_scale = 98th pct / 2.0)

    IMPORTANT — Cartopy / streamplot compatibility note:
    matplotlib.streamplot does not natively support Cartopy axes transforms.
    The implementation must use the Cartopy PlateCarree() transform workaround:
    call streamplot on the raw lat/lon arrays and then set the axes extent
    and add coastlines/gridlines separately. The streamplot call itself must
    use a standard Axes (not a GeoAxes), with coastlines drawn via
    cartopy.feature added after the fact. Alternatively, use the approach
    described in the cartopy gallery: create a plain Axes, call streamplot,
    then overlay a transparent GeoAxes for coastlines.

    Recommended implementation pattern:
        fig, ax = plt.subplots(figsize=(10, 5))
        speed = np.sqrt(U**2 + V**2)
        v_max = np.nanpercentile(speed, 98)
        lw = 0.5 + 2.0 * speed / v_max
        strm = ax.streamplot(lons, lats, vz_grid, vm_grid,
                             color=speed, cmap='plasma',
                             linewidth=lw, density=1.5,
                             arrowsize=1.2,
                             norm=plt.Normalize(0, v_max))
        plt.colorbar(strm.lines, ax=ax, label='Wind speed (m/s)')
        # Overlay coastlines via a second axis
        ax_geo = fig.add_axes(ax.get_position(), projection=ccrs.PlateCarree(),
                              frameon=False)
        ax_geo.set_extent([-180, 180, -90, 90], crs=ccrs.PlateCarree())
        ax_geo.add_feature(cfeature.COASTLINE, linewidth=0.6, edgecolor='gray')
        ax_geo.gridlines(draw_labels=False, linewidth=0.3, alpha=0.4)

    Layout: figure size (10, 5) inches.

    Physical interpretation note: streamplot excels at revealing large-scale
    circulation structure — thermospheric super-rotation, storm-driven
    vortices, and cross-equatorial flows that quiver plots obscure with
    arrow clutter. This is the recommended mode for conference posters
    (WC-A, WC-O, WC-P series).
    """
    import matplotlib.pyplot as plt
    import cartopy.crs as ccrs
    import cartopy.feature as cfeature

    from src.windmap.nb00_wind_map import GridWindMap
    lats = GridWindMap.LAT_GRID
    lons = GridWindMap.LON_GRID

    speed = np.sqrt(vz_grid**2 + vm_grid**2)
    v_max = float(np.nanpercentile(speed, 98))
    lw = 0.5 + 2.0 * speed / max(v_max, 1.0)

    fig, ax = plt.subplots(1, 1, figsize=(10, 5))
    fig.suptitle(title, fontsize=11)

    strm = ax.streamplot(
        lons, lats, vz_grid, vm_grid,
        color=speed,
        cmap='plasma',
        linewidth=lw,
        density=1.5,
        arrowsize=1.2,
        norm=plt.Normalize(0, v_max),
    )
    plt.colorbar(strm.lines, ax=ax, orientation='horizontal',
                 pad=0.05, label='Wind speed (m/s)')
    ax.set_xlim(-180, 180)
    ax.set_ylim(-90, 90)
    ax.set_xlabel('Longitude (°)')
    ax.set_ylabel('Latitude (°)')

    # Overlay coastlines via transparent GeoAxes
    ax_geo = fig.add_axes(ax.get_position(),
                          projection=ccrs.PlateCarree(),
                          frameon=False)
    ax_geo.set_extent([-180, 180, -90, 90], crs=ccrs.PlateCarree())
    ax_geo.add_feature(cfeature.COASTLINE, linewidth=0.6, edgecolor='gray')
    ax_geo.gridlines(draw_labels=False, linewidth=0.3, alpha=0.4)

    plt.tight_layout()
    if save_path:
        plt.savefig(save_path, dpi=150, bbox_inches='tight')
        plt.close(fig)
    else:
        plt.show()
```

### 3.5 `_plot_magnitude()` — speed magnitude + STM threshold contour

```python
def _plot_magnitude(
    self,
    vz_grid: np.ndarray,
    vm_grid: np.ndarray,
    title: str,
    save_path: str | None,
) -> None:
    """
    Single-panel figure: wind speed magnitude with STM threshold contour.

    Colourmap: 'hot_r' (white → yellow → red), range [0, v_max] where
    v_max = 98th percentile of the speed distribution.

    A single black contour line is drawn at WIND_BIAS_BUDGET_MS (from
    src.constants, nominally 9.8 m/s) to mark the boundary between
    regions where wind speed exceeds the pipeline's 1-sigma precision
    requirement. Contour is labelled inline.

    A second dashed contour is drawn at 100 m/s to delineate the
    transition from sub-sonic to super-sonic thermospheric flow regimes.

    Colourbar: vertical, right side, labelled 'Wind speed (m/s)'.
    Layout: single axes, figure size (10, 5) inches.

    Physical interpretation note: this mode highlights where the
    thermospheric wind is fast enough to matter for science (above the
    9.8 m/s noise floor) and where extreme winds occur (above 100 m/s).
    Useful for STM traceability figures and for identifying regions of
    interest in storm cases.
    """
    import matplotlib.pyplot as plt
    import cartopy.crs as ccrs
    import cartopy.feature as cfeature

    try:
        from src.constants import WIND_BIAS_BUDGET_MS
    except ImportError:
        WIND_BIAS_BUDGET_MS = 9.8   # fallback if constants not available

    from src.windmap.nb00_wind_map import GridWindMap
    lats = GridWindMap.LAT_GRID
    lons = GridWindMap.LON_GRID

    speed = np.sqrt(vz_grid**2 + vm_grid**2)
    v_max = float(np.nanpercentile(speed, 98))

    fig, ax = plt.subplots(
        1, 1, figsize=(10, 5),
        subplot_kw={'projection': ccrs.PlateCarree()},
    )
    fig.suptitle(title, fontsize=11)
    ax.add_feature(cfeature.COASTLINE, linewidth=0.5)
    ax.add_feature(cfeature.BORDERS,   linewidth=0.3)
    ax.gridlines(draw_labels=True, linewidth=0.3, alpha=0.5)

    im = ax.pcolormesh(lons, lats, speed,
                       vmin=0, vmax=v_max,
                       cmap='hot_r',
                       transform=ccrs.PlateCarree())
    plt.colorbar(im, ax=ax, orientation='vertical',
                 pad=0.02, label='Wind speed (m/s)')

    # STM threshold contour (solid black)
    cs1 = ax.contour(lons, lats, speed,
                     levels=[WIND_BIAS_BUDGET_MS],
                     colors='black', linewidths=1.0,
                     transform=ccrs.PlateCarree())
    ax.clabel(cs1, fmt=f'{WIND_BIAS_BUDGET_MS:.1f} m/s', fontsize=7)

    # 100 m/s contour (dashed black)
    cs2 = ax.contour(lons, lats, speed,
                     levels=[100.0],
                     colors='black', linewidths=0.8,
                     linestyles='dashed',
```

## 4. GridWindMap base class (internal)

*(Unchanged from 2026-04-06 version.)*

`GridWindMap` is a concrete partial implementation of `WindMap` that provides:
- A 180×360 lat/lon grid stored as numpy arrays
- Bilinear interpolation via `scipy.interpolate.RegularGridInterpolator`
- `to_netcdf()` / `from_netcdf()` serialisation

T2, T3, and T4 all subclass `GridWindMap`. T1 (`UniformWindMap`) does not
need a grid and subclasses `WindMap` directly.

```python
from scipy.interpolate import RegularGridInterpolator

class GridWindMap(WindMap):
    """
    WindMap backed by a 180×360 lat/lon grid with bilinear interpolation.

    Subclasses must populate self.v_zonal_grid and self.v_merid_grid
    (both shape (180, 360), float32) in their __init__ before calling
    super().__init__() — or call _build_interpolators() explicitly
    after populating the grids.

    Grid layout:
        lat_grid[i] = -89.5 + i   degrees, i in 0..179
        lon_grid[j] = -179.5 + j  degrees, j in 0..359
    """

    LAT_GRID = np.linspace(-89.5, 89.5, 180)   # cell centres, degrees
    LON_GRID = np.linspace(-179.5, 179.5, 360)  # cell centres, degrees

    def _build_interpolators(self) -> None:
        """Build scipy RegularGridInterpolator objects from populated grids."""
        self._interp_zonal = RegularGridInterpolator(
            (self.LAT_GRID, self.LON_GRID),
            self.v_zonal_grid,
            method='linear',
            bounds_error=False,
            fill_value=None,   # extrapolate at poles
        )
        self._interp_merid = RegularGridInterpolator(
            (self.LAT_GRID, self.LON_GRID),
            self.v_merid_grid,
            method='linear',
            bounds_error=False,
            fill_value=None,
        )

    def sample(self, lat_deg: float, lon_deg: float) -> tuple[float, float]:
        """Bilinear interpolation at (lat_deg, lon_deg)."""
        lon_deg = ((lon_deg + 180.0) % 360.0) - 180.0
        lat_deg = float(np.clip(lat_deg, -90.0, 90.0))
        pt = np.array([[lat_deg, lon_deg]])
        vz = float(self._interp_zonal(pt))
        vm = float(self._interp_merid(pt))
        return (vz, vm)

    def to_netcdf(self, filepath: str, metadata: dict | None = None) -> None:
        """
        Save this wind map to a NetCDF4 file.
        [Schema unchanged from 2026-04-06 version — see that spec for full details.]
        """

    @classmethod
    def from_netcdf(cls, filepath: str) -> 'GridWindMap':
        """
        Load a previously saved GridWindMap from a NetCDF4 file.
        [Unchanged from 2026-04-06 version.]
        """
```

## 12. External validation reference

*(Unchanged from 2026-04-06 version.)*

For visual and numerical validation of T3 and T4 output, compare against:

- **HWM14 paper:** Drob et al. (2015), *Earth and Space Science*, 2,
  doi:10.1002/2014EA000089. Figures 4–6 show zonal and meridional wind maps
  at 250 km for quiet conditions — the pattern should qualitatively match
  the T3 output.
- **ICON/MIGHTI L3 winds:** NASA Earthdata, ICON mission. At 250 km,
  quiet-time zonal winds of 100–300 m/s eastward at low-to-mid latitudes are
  typical. Download via `earthdata.nasa.gov` (search "ICON_L3-7_MIGHTI").
- **TIE-GCM:** For T4 storm validation, TIE-GCM output is publicly accessible
  via the CEDAR Madrigal database (`cedar.openmadrigal.org`).
