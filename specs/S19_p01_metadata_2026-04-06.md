## 1. Purpose

P01 defines the canonical metadata representation for every WindCube FPI
image — dark, calibration, or science — whether real (ingested from the
on-orbit binary) or synthetic (generated by the forward modelling pipeline).

Three deliverables:

1. **`ImageMetadata` dataclass** — the single in-memory representation used
   by all pipeline modules. Every module that processes an image receives an
   `ImageMetadata` alongside the pixel array.

2. **Level-0 sidecar JSON schema** — one `.json` file per `.bin` file,
   written at ingest time, containing the complete `ImageMetadata` record.
   This is the persistent metadata store for real images. For synthetic
   images the sidecar is optionally written by the synthesis pipeline.

3. **Ingest functions** — two public functions: one for real binary images
   (`ingest_real_image`) and one for synthetic pipeline outputs
   (`build_synthetic_metadata`).

**What this module is not.** P01 does not write pixel data to disk, does not
define the Level-1 or Level-2 results sidecar (those belong to S20), and
does not implement any scientific analysis. It is purely a data
representation and I/O layer.

**Sidecar boundary.** The Level-0 sidecar defined here carries pre-processing
metadata only — instrument state, attitude, orbit position, timing, and
provenance. Post-processing results (recovered `epsilon_cal`, `v_rel_ms`,
`v_zonal`, `v_meridional`, fit quality flags) belong in a separate Level-2
sidecar defined in S20. The two files share the same stem but different
suffixes: `<stem>_L0.json` (this spec) and `<stem>_L2.json` (S20).

## 2. Binary format reference

*Source: `load_real_image.py`, empirically verified against `*_analyzed.txt`
files. All values match the XCAM camera documentation.*

### 2.1 File geometry

```
Total size : 260 rows × 276 cols × 2 bytes = 143,520 bytes
Row 0      : 276 uint16 words — embedded metadata header
Rows 1–259 : 259 × 276 uint16 — image pixels (14-bit ADC; range 0–16383)
Endianness : big-endian uint16 throughout
```

The `ingest_real_image` function must validate file size before parsing.
Any deviation from 143,520 bytes raises `ValueError` immediately.

### 2.2 Mixed-endian encoding for multi-byte fields

Multi-byte fields (uint64 timestamps, float64 scalars and vectors) use a
**mixed-endian** encoding: each 64-bit value is stored as four consecutive
big-endian uint16 words in **little-endian word order** (least-significant
word first).

Decode algorithm:

```python
def _u64(h: np.ndarray, w: int) -> int:
    """4 BE uint16 words in LE word order → uint64."""
    return sum(int(h[w + i]) << (16 * i) for i in range(4))

def _f64(h: np.ndarray, w: int) -> float:
    """4 BE uint16 words in LE word order → float64."""
    import struct
    b = struct.pack(">4H", *reversed([h[w + i] for i in range(4)]))
    return struct.unpack(">d", b)[0]

def _u8arr(h: np.ndarray, w: int, n: int) -> list[int]:
    """n consecutive uint16 words → list of uint8 from low byte."""
    return [int(h[w + i]) & 0xFF for i in range(n)]
```

### 2.3 Header word map (row 0, words 0–275)

| Words | Field name (binary) | Type | Units | Notes |
|-------|--------------------|----|-------|-------|
| 0 | `rows` | uint16 | px | Always 260 (incl. header row) |
| 1 | `cols` | uint16 | px | Always 276 (2×2 binned) |
| 2 | `exp_time` | uint16 | cs | Centiseconds (1/100 s) |
| 3 | `exp_unit` | uint16 | reg | CCD timing register |
| 4–7 | `ccd_temp1` | float64 | °C | On-chip thermistor |
| 8–11 | `lua_timestamp` | uint64 | ms | Unix epoch, end of exposure |
| 12–15 | `adcs_timestamp` | uint64 | ms | ADCS clock; 0 = unavailable |
| 16–19 | `lat_hat` | float64 | rad | SC geodetic latitude |
| 20–23 | `lon_hat` | float64 | rad | SC geodetic longitude |
| 24–27 | `alt_hat` | float64 | m | SC altitude above WGS84 |
| 28–43 | `ads_q_hat[4]` | float64×4 | — | Binary order: [w,x,y,z] |
| 44–59 | `acs_q_err[4]` | float64×4 | — | Binary order: [w,x,y,z] |
| 60–71 | `pos_eci_hat[3]` | float64×3 | m | ECI position [x,y,z] |
| 72–83 | `vel_eci_hat[3]` | float64×3 | m/s | ECI velocity [vx,vy,vz] |
| 84–99 | `b2_temp_f[4]` | float64×4 | °C | Etalon temps [T0,T1,T2,T3] |
| 100–103 | `gpio_pwr_on[4]` | uint8×4 | — | Low byte of each uint16 |
| 104–109 | `lamp_ch_on[6]` | uint8×6 | — | Low byte; channels 0–5 |
| 110–275 | — | — | — | Padding / reserved |

### 2.4 Quaternion re-ordering

The binary header stores quaternions in `[w, x, y, z]` (scalar-first) order.
All pipeline internals and the JSON sidecar use `[x, y, z, w]` (scalar-last).
The ingest function performs this re-ordering automatically:

```python
q_wxyz = [_f64(h, 28 + i * 4) for i in range(4)]
attitude_quaternion = [q_wxyz[1], q_wxyz[2], q_wxyz[3], q_wxyz[0]]  # → [x,y,z,w]
```

Both `ads_q_hat` (attitude quaternion) and `acs_q_err` (pointing error
quaternion) undergo this re-ordering.

### 2.5 Image type classification

Derived from lamp and shutter state:

```python
any_lamp_on = any(lamp_ch_array)
shutter_closed = (gpio_pwr_on[0] == 1 and gpio_pwr_on[3] == 1)

if any_lamp_on:
    img_type = "cal"
elif shutter_closed:
    img_type = "dark"
else:
    img_type = "science"
```

### 2.6 Binning derivation

```python
binning = 1 if cols == 552 else 2  # 276 → 2×2 binned; 552 → 1×1 unbinned
```

### 2.7 Known typo in legacy loader

The original `load_real_image.py` contains a typo: `"attitude_quadternion"`
(extra 'd'). The canonical field name in this spec and all pipeline code is
`"attitude_quaternion"`. Any existing JSON sidecar files written by the
legacy loader must be normalised at ingest time:

```python
if "attitude_quadternion" in d:
    d["attitude_quaternion"] = d.pop("attitude_quadternion")
```

## 3. ImageMetadata dataclass

This is the single canonical in-memory representation. Every module that
receives an image also receives an `ImageMetadata`.

```python
from __future__ import annotations
from dataclasses import dataclass, field
from typing import Optional


@dataclass
class ImageMetadata:
    """
    Complete metadata record for one WindCube FPI image.

    Applies to all image types: dark, cal, science.
    Applies to both real (on-orbit) and synthetic images.
    Synthetic-only fields are None for real images.
    Pipeline-added fields are None at ingest and populated during processing.
    """

    # ── Section 3.1: Image geometry and classification ──────────────────────
    rows:         int    # Total rows including header row (on-orbit: 260)
    cols:         int    # Total columns (on-orbit: 276 for 2×2 binned)
    exp_time:     int    # Exposure duration, centiseconds
    exp_unit:     int    # CCD timing register value
    binning:      int    # 1 = 1×1, 2 = 2×2; derived from cols
    img_type:     str    # 'cal', 'dark', or 'science'

    # ── Section 3.2: Timing ─────────────────────────────────────────────────
    lua_timestamp:   int    # Unix epoch ms, end of exposure (primary reference)
    adcs_timestamp:  int    # ADCS clock Unix ms; 0 if unavailable
    utc_timestamp:   str    # ISO 8601 string derived from lua_timestamp

    # ── Section 3.3: Orbit state ────────────────────────────────────────────
    spacecraft_latitude:   float         # SC geodetic latitude, rad
    spacecraft_longitude:  float         # SC geodetic longitude, rad
    spacecraft_altitude:   float         # SC altitude above WGS84, m
    pos_eci_hat:  list[float]            # ECI position [x, y, z], m
    vel_eci_hat:  list[float]            # ECI velocity [vx, vy, vz], m/s

    # ── Section 3.4: Attitude ───────────────────────────────────────────────
    attitude_quaternion:  list[float]    # [x, y, z, w] scalar-last (pipeline convention)
    pointing_error:       list[float]    # [x, y, z, w] residual attitude error
    obs_mode:             str            # 'along_track', 'cross_track', or 'unknown'

    # ── Section 3.5: Instrument state ───────────────────────────────────────
    ccd_temp1:      float                # CCD temperature, °C
    etalon_temps:   list[float]          # [T0, T1, T2, T3], °C
    shutter_status: str                  # 'open' or 'closed'
    gpio_pwr_on:    list[int]            # GPIO register values [ch0–ch3]
    lamp_ch_array:  list[int]            # Lamp channel states [0–5]
    lamp1_status:   str                  # 'on' or 'off'
    lamp2_status:   str                  # 'on' or 'off'
    lamp3_status:   str                  # 'on' or 'off'

    # ── Section 3.6: Orbit and sequence identification ──────────────────────
    orbit_number:   Optional[int] = None  # Absolute orbit counter (0-based).
                                          # Derived from lua_timestamp + TLE at
                                          # ingest time if TLE is available;
                                          # None otherwise. Set explicitly by
                                          # the synthesis pipeline.
    frame_sequence: Optional[int] = None  # Frame index within this orbit
                                          # (0-based, reset at each orbit
                                          # boundary). Set by the SOC ingestion
                                          # scheduler or synthesis pipeline.
    orbit_parity:   Optional[str]  = None # 'along_track' (odd orbits) or
                                          # 'cross_track' (even orbits).
                                          # Redundant with obs_mode but adds
                                          # the absolute orbit context needed
                                          # by M07 pairing. None if
                                          # orbit_number is None.

    # ── Section 3.7: Pointing quality gate ──────────────────────────────────
    adcs_quality_flag: int = 0            # Bitmask. See AdcsQualityFlags.
                                          # 0 = GOOD. Populated at ingest
                                          # from pointing_error magnitude
                                          # and ADCS mode telemetry.

    # ── Section 3.8: Dark subtraction provenance ────────────────────────────
    dark_subtracted:      bool = False    # True once M03 subtract_dark() applied
    dark_n_frames:        int  = 0        # Number of dark frames in master dark
    dark_lua_timestamp:   Optional[int] = None  # lua_timestamp of the dark
                                                 # frame(s) used; None if not
                                                 # yet subtracted
    dark_etalon_temp_mean: Optional[float] = None  # Mean etalon temperature
                                                    # at dark acquisition, °C

    # ── Section 4: Synthetic fields ─────────────────────────────────────────
    is_synthetic:       bool  = False
    truth_v_los:        Optional[float] = None   # Ground-truth LOS wind, m/s
    truth_v_zonal:      Optional[float] = None   # Ground-truth zonal wind, m/s
    truth_v_meridional: Optional[float] = None   # Ground-truth meridional wind, m/s
    tangent_lat:        Optional[float] = None   # Tangent point latitude, deg
    tangent_lon:        Optional[float] = None   # Tangent point longitude, deg
    tangent_alt_km:     Optional[float] = None   # Tangent altitude, km
    etalon_gap_mm:      Optional[float] = None   # Etalon gap used, mm
    noise_seed:         Optional[int]   = None   # Random seed for noise generation

    # ── Section 5: Etalon thermal correction hook ───────────────────────────
    etalon_gap_corrected_mm: Optional[float] = None
    # Thermally corrected etalon gap, mm. Computed as:
    #   d_corrected = d_ref + k_thermal × (T_etalon_mean - T_ref)
    # where:
    #   d_ref      = ETALON_GAP_M × 1000  (from S03, mm)
    #   T_etalon_mean = mean(etalon_temps)
    #   T_ref      = etalon temperature at most recent calibration acquisition
    #   k_thermal  = TBD (awaiting CTE measurement from opto-mechanical engineer)
    #
    # This field is None at ingest and populated by the calibration scheduler
    # after the preceding cal frame's etalon_temps are known. M05 should prefer
    # this value over the static ETALON_GAP_M when it is not None.
    #
    # Physical rationale: the etalon gap shifts the fringe phase reference.
    # A 1 °C temperature change at typical invar CTE (~1.5 ppm/°C) and
    # 20 mm gap produces a gap change of ~30 nm ≈ 0.094 FSR, equivalent to
    # a ~445 m/s wind bias. Even a rough CTE value tightens this substantially.

    # ── Section 6: Grafana telemetry hook ───────────────────────────────────
    grafana_record_id: Optional[str] = None
    # TBD — unique record ID linking this image to its corresponding row in
    # the Grafana/InfluxDB housekeeping telemetry database. When populated,
    # allows retrieval of additional housekeeping channels (reaction wheel
    # speeds, battery voltage, ADCS mode history) not captured in the binary
    # header. Format TBD pending Grafana schema definition.
    # Populated by the SOC ingestion scheduler; None for synthetic images.
```

## 4. AdcsQualityFlags bitmask

```python
class AdcsQualityFlags:
    """
    Bitmask for adcs_quality_flag in ImageMetadata.
    Multiple flags may be set simultaneously.
    """
    GOOD             = 0x00  # No issues detected
    SLEW_IN_PROGRESS = 0x01  # pointing_error magnitude > 30 arcsec RMS
                              # (SYS.108 AKE budget from SI-UCAR-WC-RP-004)
                              # Image acquired during attitude manoeuvre;
                              # wind retrieval unreliable
    STR_UNAVAILABLE  = 0x02  # Star tracker not contributing to attitude solution.
                              # ADCS in Coarse Pointing mode, not Fine Pointing.
                              # AKE likely degraded beyond 30 arcsec.
    GNSS_UNAVAILABLE = 0x04  # GNSS position/velocity invalid at acquisition time.
                              # THRF guidance error grows; tangent point
                              # geolocation uncertain.
    ADCS_DEGRADED    = 0x08  # Generic ADCS degradation not covered by above flags.
                              # Set when ADCS telemetry indicates anomalous state.
    POINTING_UNKNOWN = 0x10  # adcs_quality_flag cannot be determined — telemetry
                              # missing or timestamp mismatch. Treat as degraded.
```

**Pointing quality gate rule for wind retrieval modules (M05, M06, M07):**
Any image with `adcs_quality_flag != 0` should be flagged in the processing
result. Images with `SLEW_IN_PROGRESS` set should be excluded from wind
retrieval entirely. Images with `STR_UNAVAILABLE` or `GNSS_UNAVAILABLE` may
be processed but results should carry a degraded-confidence flag.

**Computing `adcs_quality_flag` at ingest:**

```python
def compute_adcs_quality_flag(meta: dict) -> int:
    """
    Compute the ADCS quality bitmask from raw parsed metadata.

    Parameters
    ----------
    meta : dict from parse_header() — contains pointing_error as [x,y,z,w]

    Returns
    -------
    int : AdcsQualityFlags bitmask
    """
    import numpy as np
    flag = AdcsQualityFlags.GOOD

    # Pointing error magnitude — convert quaternion to angle (arcsec)
    qe = meta.get("pointing_error", [0.0, 0.0, 0.0, 1.0])
    # |q_err| → rotation angle: 2 × arcsin(|[x,y,z]|), in arcsec
    vec_norm = np.sqrt(qe[0]**2 + qe[1]**2 + qe[2]**2)
    vec_norm = min(vec_norm, 1.0)   # clamp for numerical safety
    angle_arcsec = 2.0 * np.degrees(np.arcsin(vec_norm)) * 3600.0

    AKE_BUDGET_ARCSEC = 30.0   # SYS.108 from SI-UCAR-WC-RP-004
    if angle_arcsec > AKE_BUDGET_ARCSEC:
        flag |= AdcsQualityFlags.SLEW_IN_PROGRESS

    # GNSS availability — pos_eci_hat == [0,0,0] indicates no valid fix
    pos = meta.get("pos_eci_hat", [1.0, 0.0, 0.0])
    if all(p == 0.0 for p in pos):
        flag |= AdcsQualityFlags.GNSS_UNAVAILABLE

    # adcs_timestamp == 0 means ADCS telemetry was not available
    if meta.get("adcs_timestamp", 0) == 0:
        flag |= AdcsQualityFlags.ADCS_DEGRADED

    return flag
```

## 5. Ingest functions

### 5.1 Real image ingest

```python
def ingest_real_image(
    path: str | pathlib.Path,
    obs_mode: str = "unknown",
    orbit_number: Optional[int] = None,
    frame_sequence: Optional[int] = None,
) -> tuple[ImageMetadata, np.ndarray]:
    """
    Load a WindCube FPI binary image and parse its metadata header.

    Parameters
    ----------
    path : str or Path
        Path to the `*_swapped.bin` file.
    obs_mode : str
        'along_track', 'cross_track', or 'unknown'. Must be supplied by the
        caller from the mission schedule — it cannot be determined from the
        binary header alone.
    orbit_number : int, optional
        Absolute orbit counter. Supply from the SOC ingestion scheduler
        if available; None otherwise.
    frame_sequence : int, optional
        Frame index within this orbit. Supply from scheduler; None otherwise.

    Returns
    -------
    (ImageMetadata, np.ndarray)
        The metadata record and the pixel array (259 × 276, uint16).
        No files are written. No ROI extraction or masking is performed.
        Those are M03's responsibility.

    Raises
    ------
    ValueError
        If file size != 143,520 bytes.
    FileNotFoundError
        If path does not exist.

    Notes
    -----
    The pixel array contains rows 1–259 of the binary file (the image region).
    Row 0 (the header) is consumed by this function and not returned in the
    pixel array. The 14-bit ADU range is 0–16383; values > 16383 indicate
    a saturated or corrupt pixel.
    """
```

Implementation steps (in order):
1. Validate file size: `expected = 260 × 276 × 2 = 143520 bytes`
2. Load with `np.frombuffer(..., dtype=">u2")` — big-endian uint16
3. Split: `header = raw[:276]`, `image = raw[276:].reshape(259, 276)`
4. Call `parse_header(header)` to get raw field dict
5. Normalise legacy typo (`attitude_quadternion` → `attitude_quaternion`)
6. Derive `binning`, `img_type`, `shutter_status` per Section 2
7. Derive `utc_timestamp` from `lua_timestamp`
8. Compute `adcs_quality_flag` via `compute_adcs_quality_flag()`
9. Derive `orbit_parity` from `orbit_number % 2` if `orbit_number` is not None
   (0 = even = cross_track per WC-SE-0003; 1 = odd = along_track)
10. Construct and return `ImageMetadata` with all real fields populated,
    all synthetic fields set to `None`, `is_synthetic=False`

## 6. JSON sidecar schema

### 6.1 File naming convention

```
<stem>_L0.json    ← Level-0 metadata sidecar (this spec)
<stem>_L2.json    ← Level-2 results sidecar (S20)
```

Where `<stem>` is the binary filename without its `.bin` extension.
Example: `WC_20250315T023107_orbit0042_f0017_L0.json`.

### 6.2 Formatting rules

- **Encoding:** UTF-8, no BOM
- **Floats:** Full double precision; no rounding
- **Integers:** JSON number type; `lua_timestamp` must not be cast to float
  (53-bit float mantissa is insufficient for millisecond Unix timestamps
  beyond year 2255 — use integer)
- **Nulls:** JSON `null`; every key defined in the schema must be present
- **Booleans:** JSON `true` / `false`
- **Arrays:** JSON arrays `[...]`; no numpy types at serialisation time

### 6.3 Serialisation / deserialisation helpers

```python
def write_sidecar(meta: ImageMetadata, path: str | pathlib.Path) -> None:
    """Write ImageMetadata to a Level-0 JSON sidecar file."""
    import json
    import dataclasses
    d = dataclasses.asdict(meta)
    with open(path, "w", encoding="utf-8") as f:
        json.dump(d, f, indent=2, ensure_ascii=False)


def read_sidecar(path: str | pathlib.Path) -> ImageMetadata:
    """
    Read a Level-0 JSON sidecar and return an ImageMetadata.
    Applies legacy typo normalisation automatically.
    """
    import json
    with open(path, encoding="utf-8") as f:
        d = json.load(f)
    # Legacy normalisation
    if "attitude_quadternion" in d:
        d["attitude_quaternion"] = d.pop("attitude_quadternion")
    return ImageMetadata(**d)
```
