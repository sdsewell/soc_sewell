## 1. Purpose and design intent

Z02 is a **standalone interactive Python script** (not a library module).
The user runs it from the command line, is prompted for three instrument
parameters and a save location, and the script produces:

1. A synthetic 2D airglow fringe image synthesised via M01/M04 physics.
2. A complete S19-compliant binary header embedded in row 0 of the image.
3. A `.bin` file saved to a user-selected folder (Windows path dialog or
   typed path).
4. A printed parameter table summarising all instrument and metadata values.

The output file is byte-for-byte compatible with `ingest_real_image()` from
P01 — the Z01 validation script can load it directly as if it were a real
on-orbit image.

**What Z02 does not do:**
- Z02 does not implement any new physics. It calls M01/M04 for image
  synthesis.
- Z02 does not write a JSON sidecar. The header IS the metadata, as in
  real on-orbit files. A sidecar may optionally be written if time permits
  (Section 7, optional extension).
- Z02 does not perform dark subtraction or any reduction steps.

## 2. Physical background — image synthesis

### 2.1 From instrument parameters to a fringe image

The synthetic airglow fringe image is generated by the following chain,
all of which is already implemented in M01 and M04:

```
InstrumentParams(t, R_refl, alpha)
        ↓
  v_rel_to_lambda_c(v_rel=0 m/s)  →  λ_c = OI_WAVELENGTH_M (zero wind)
        ↓
  airy_modified(r, λ_c, params)   →  1D radial profile S(r)
        ↓
  radial_profile_to_image(S, r)   →  2D image (image_size × image_size)
        ↓
  add_gaussian_noise(image, snr)  →  noisy 2D image
```

The user controls:
- **`t` (etalon gap, mm):** determines fringe spacing via FSR = λ²/(2t).
  Default: 20.106 mm (operational Tolansky value; NOT the ICOS spacer 20.008 mm).
- **`R_refl` (reflectivity):** determines finesse and fringe contrast.
  Default: 0.53.
- **`f_lens` (focal length, mm):** determines plate scale α = pixel_size / f_lens
  in rad/px. Default: 199.1 mm (Tolansky result). Pixel size = 16 µm × 2 (2×2
  binned) = 32 µm.

### 2.2 Plate scale (alpha) derived from focal length

The magnification constant α maps pixel radius to angle:

```
pixel_size_binned = CCD_PIXEL_SIZE_UM × 2 = 16 µm × 2 = 32 µm = 32e-6 m
α = pixel_size_binned / f_lens   (in rad/px, small-angle approximation)

Default: α = 32e-6 / 0.1991 = 1.6072e-4 rad/px  ✓ (matches Tolansky result)
```

The user sets `f_lens`; α is computed internally.

### 2.3 Image dimensions and binary format

Z02 generates images in the **exact on-orbit binary format** defined in S19 §2:

```
Total file: 260 rows × 276 cols × 2 bytes = 143,520 bytes
Row 0:      276 uint16 words — metadata header (constructed by Z02)
Rows 1–259: 259 × 276 uint16 — image pixels (14-bit ADU)
```

The synthetic airglow image from M04 is 256×256. Z02 embeds it at the
centre of the 259×276 pixel region, zero-padding rows and columns as
needed. This matches the real on-orbit layout where the active region is
centred in the CCD array.

**Image embedding rule:**
```
pixel_block[r_start:r_start+256, c_start:c_start+256] = airglow_image_uint16
where:
  r_start = (259 - 256) // 2 = 1   (top padding rows)
  c_start = (276 - 256) // 2 = 10  (left padding columns)
```

### 2.4 Pixel scaling (float counts → uint16 ADU)

M04 returns float64 counts. The binary format stores 14-bit ADU (0–16383).
Scale rule:

```
adu = np.clip(np.round(counts * scale_factor), 0, 16383).astype(np.uint16)
```

`scale_factor` is computed so that `I0` (default 1000 counts) maps to ADU
4000, leaving headroom below 16383 for noise:

```
scale_factor = 4.0   (i.e. 1 count → 4 ADU; I0=1000 → ADU 4000)
```

## 3. Metadata header construction

### 3.1 Design principle

Z02 writes a **complete, valid S19-compliant header** into row 0 of the
`.bin` file. For fields with no meaningful synthetic value, zero is written.
The etalon temperature is set to `24.0 °C` (specified). The image type is
always `"science"` (no lamp channels active, shutter open).

### 3.2 Encoding functions

All encoding must use the mixed-endian scheme from S19 §2.2. The inverse
of the decode functions defined there:

```python
def _encode_u16(h: np.ndarray, w: int, val: int) -> None:
    """Write a uint16 value to word w."""
    h[w] = int(val) & 0xFFFF

def _encode_u64(h: np.ndarray, w: int, val: int) -> None:
    """Encode uint64 as 4 BE uint16 words in LE word order (S19 §2.2)."""
    for i in range(4):
        h[w + i] = (int(val) >> (16 * i)) & 0xFFFF

def _encode_f64(h: np.ndarray, w: int, val: float) -> None:
    """Encode float64 as 4 BE uint16 words in LE word order (S19 §2.2)."""
    import struct
    b = struct.pack(">d", val)
    words = struct.unpack(">4H", b)
    for i in range(4):
        h[w + i] = words[3 - i]   # LE word order: LSW first
```

### 3.3 Header word map — Z02 field assignments

All word positions from S19 §2.3. Fields not listed use zero.

| Words | Field | Z02 value | Notes |
|-------|-------|-----------|-------|
| 0 | `rows` | 260 | Always |
| 1 | `cols` | 276 | Always (2×2 binned) |
| 2 | `exp_time` | 6000 | 60 s in centiseconds (representative science exposure) |
| 3 | `exp_unit` | 1 | Register value |
| 4–7 | `ccd_temp1` | 0.0 | float64; zero (no hardware) |
| 8–11 | `lua_timestamp` | current Unix time in ms | uint64; `int(time.time() * 1000)` |
| 12–15 | `adcs_timestamp` | 0 | uint64; zero |
| 16–19 | `lat_hat` | 0.0 | float64; zero |
| 20–23 | `lon_hat` | 0.0 | float64; zero |
| 24–27 | `alt_hat` | 510000.0 | float64; 510 km (nominal SSO altitude) |
| 28–43 | `ads_q_hat[4]` | [1.0, 0.0, 0.0, 0.0] (wxyz) | Identity quaternion (scalar-first in binary) |
| 44–59 | `acs_q_err[4]` | [1.0, 0.0, 0.0, 0.0] (wxyz) | Identity quaternion |
| 60–71 | `pos_eci_hat[3]` | [0.0, 6891000.0, 0.0] | float64×3; plausible SSO position |
| 72–83 | `vel_eci_hat[3]` | [7560.0, 0.0, 0.0] | float64×3; plausible SSO velocity |
| 84–99 | `b2_temp_f[4]` | [24.0, 24.0, 24.0, 24.0] | **Etalon temp = 24.0 °C** (specified) |
| 100–103 | `gpio_pwr_on[4]` | [0, 0, 0, 0] | uint8; shutter open (science image) |
| 104–109 | `lamp_ch_on[6]` | [0, 0, 0, 0, 0, 0] | uint8; all lamps off (science) |
| 110–275 | reserved | 0 | padding |

**Quaternion encoding note:** The binary header uses scalar-first `[w, x, y, z]`
order (S19 §2.4). Z02 must write the identity quaternion as `[w=1, x=0, y=0, z=0]`
at the appropriate words for both `ads_q_hat` (words 28–43) and `acs_q_err`
(words 44–59). When P01's `ingest_real_image()` reads this back, it will
re-order to `[x=0, y=0, z=0, w=1]` (scalar-last pipeline convention).

### 4.2 `_encode_quaternion_wxyz`

```python
def _encode_quaternion_wxyz(
    h: np.ndarray,
    w_start: int,
    q_wxyz: list[float],
) -> None:
    """
    Encode a quaternion in [w, x, y, z] scalar-first order into the header.
    Each component is encoded as a float64 using _encode_f64.
    Occupies 4 × 4 = 16 words starting at w_start.

    Parameters
    ----------
    h       : header array, shape (276,), dtype '>u2'
    w_start : first word index (28 for ads_q_hat; 44 for acs_q_err)
    q_wxyz  : [w, x, y, z], the scalar-first quaternion to encode
    """
    for i, component in enumerate(q_wxyz):
        _encode_f64(h, w_start + i * 4, component)
```

### 4.3 `build_header`

```python
def build_header(
    t_mm: float,
    etalon_temp_c: float = 24.0,
    exp_time_cs: int = 6000,
) -> np.ndarray:
    """
    Build a 276-word S19-compliant metadata header for a synthetic science image.

    Parameters
    ----------
    t_mm          : etalon gap in mm; stored for provenance (no header word
                    for this, so it is printed only — see Section 5).
    etalon_temp_c : etalon temperature in °C. Stored in b2_temp_f[0–3] (words
                    84–99). Specified value: 24.0 °C.
    exp_time_cs   : exposure time in centiseconds. Default: 6000 (60 s).

    Returns
    -------
    h : np.ndarray, shape (276,), dtype '>u2' (big-endian uint16)

    Notes
    -----
    lua_timestamp is set from time.time() at call time so each generated
    file gets a unique, monotonically increasing timestamp.
    All zero-valued fields are explicitly zero (no implicit numpy zeros).
    """
```

Implementation:
```python
    import time, struct
    h = np.zeros(276, dtype=">u2")

    # Geometry
    _encode_u16(h, 0, 260)          # rows
    _encode_u16(h, 1, 276)          # cols
    _encode_u16(h, 2, exp_time_cs)  # exp_time
    _encode_u16(h, 3, 1)            # exp_unit

    # CCD temperature (zero — no hardware)
    _encode_f64(h, 4, 0.0)

    # Timestamps
    lua_ts = int(time.time() * 1000)
    _encode_u64(h, 8,  lua_ts)
    _encode_u64(h, 12, 0)           # adcs_timestamp = 0

    # Orbit state (zeros for lat/lon; plausible altitude, pos, vel)
    _encode_f64(h, 16, 0.0)         # lat_hat
    _encode_f64(h, 20, 0.0)         # lon_hat
    _encode_f64(h, 24, 510_000.0)   # alt_hat, m

    # Attitude — identity quaternion [w=1, x=0, y=0, z=0] (scalar-first)
    _encode_quaternion_wxyz(h, 28, [1.0, 0.0, 0.0, 0.0])  # ads_q_hat
    _encode_quaternion_wxyz(h, 44, [1.0, 0.0, 0.0, 0.0])  # acs_q_err

    # ECI position and velocity (plausible SSO orbit)
    _encode_f64(h, 60, 0.0);     _encode_f64(h, 64, 6_891_000.0); _encode_f64(h, 68, 0.0)
    _encode_f64(h, 72, 7_560.0); _encode_f64(h, 76, 0.0);          _encode_f64(h, 80, 0.0)

    # Etalon temperatures — all four channels = etalon_temp_c
    for i in range(4):
        _encode_f64(h, 84 + i * 4, etalon_temp_c)

    # GPIO and lamp channels — all zero (science image, shutter open, lamps off)
    for w in range(100, 110):
        h[w] = 0

    # Words 110–275: reserved, remain zero
    return h
```

### 4.4 `synthesise_and_embed`

```python
def synthesise_and_embed(
    t_m: float,
    R_refl: float,
    f_lens_m: float,
    v_rel_ms: float = 0.0,
    snr: float = 5.0,
    image_size: int = 256,
    rng: np.random.Generator = None,
) -> tuple[np.ndarray, np.ndarray, dict]:
    """
    Synthesise a 2D airglow image and embed it into a 260×276 binary frame.

    Steps:
    1. Compute alpha from f_lens_m and pixel size.
    2. Construct InstrumentParams with (t_m, R_refl, alpha).
    3. Call synthesise_airglow_image() → dict with 'image_2d' (256×256 float64).
    4. Scale float counts → uint16 ADU.
    5. Create 259×276 zero pixel block; embed 256×256 image at centre.
    6. Build 276-word header via build_header().
    7. Stack header row + pixel block → (260, 276) uint16 array.

    Parameters
    ----------
    t_m        : etalon gap, metres
    R_refl     : plate reflectivity (dimensionless, 0–1)
    f_lens_m   : imaging lens focal length, metres
    v_rel_ms   : LOS wind speed, m/s (default 0 — zero wind)
    snr        : target SNR for Gaussian noise (default 5.0)
    image_size : synthesised image side length in pixels (default 256)
    rng        : numpy random Generator for reproducible noise (optional)

    Returns
    -------
    (frame_260x276, pixel_block_259x276, synthesis_result_dict)

    frame_260x276 : np.ndarray shape (260, 276), dtype '>u2'
        The complete binary-format frame ready to write to disk.
    pixel_block_259x276 : np.ndarray shape (259, 276), dtype np.uint16
        The pixel region only (rows 1–259), for display and verification.
    result_dict : dict
        Output from synthesise_airglow_image() — includes 'image_2d',
        'profile_1d', 'r_grid', 'lambda_c_m', 'snr_actual'.
    """
```

Implementation sketch:
```python
    # --- Step 1: derive alpha ---
    PIXEL_SIZE_BINNED_M = 32e-6   # 16 µm × 2 binning
    alpha = PIXEL_SIZE_BINNED_M / f_lens_m

    # --- Step 2: InstrumentParams ---
    from fpi.m01_airy_forward_model import InstrumentParams
    params = InstrumentParams(t=t_m, R_refl=R_refl, alpha=alpha)

    # --- Step 3: synthesise ---
    from fpi.m04_airglow_synthesis import synthesise_airglow_image
    if rng is None:
        rng = np.random.default_rng(42)
    result = synthesise_airglow_image(
        v_rel_ms=v_rel_ms, params=params, snr=snr,
        image_size=image_size, add_noise=True, rng=rng
    )
    img_float = result['image_2d']   # (256, 256) float64, counts

    # --- Step 4: float → uint16 ADU ---
    SCALE_FACTOR = 4.0
    img_adu = np.clip(np.round(img_float * SCALE_FACTOR), 0, 16383).astype(np.uint16)

    # --- Step 5: embed in 259×276 block ---
    pixel_block = np.zeros((259, 276), dtype=np.uint16)
    r_start = (259 - image_size) // 2   # = 1
    c_start = (276 - image_size) // 2   # = 10
    pixel_block[r_start:r_start + image_size, c_start:c_start + image_size] = img_adu

    # --- Step 6: build header ---
    header_row = build_header(t_mm=t_m * 1000)   # shape (276,), dtype '>u2'

    # --- Step 7: assemble 260×276 frame ---
    frame = np.zeros((260, 276), dtype=">u2")
    frame[0, :] = header_row
    frame[1:, :] = pixel_block.astype(">u2")

    return frame, pixel_block, result
```

### 4.5 `save_bin_file`

```python
def save_bin_file(
    frame: np.ndarray,
    output_path: pathlib.Path,
) -> None:
    """
    Write the 260×276 frame to disk as a big-endian binary file.

    The frame must be dtype '>u2' (big-endian uint16). Writing uses
    frame.astype(">u2").tobytes() to ensure correct byte order regardless
    of host endianness.

    Parameters
    ----------
    frame       : shape (260, 276), dtype '>u2'
    output_path : pathlib.Path for output .bin file

    Raises
    ------
    ValueError if frame.shape != (260, 276)
    IOError   if directory does not exist or is not writable
    """
    if frame.shape != (260, 276):
        raise ValueError(f"frame.shape {frame.shape} != (260, 276)")
    output_path.parent.mkdir(parents=True, exist_ok=True)
    frame.astype(">u2").tofile(output_path)
    expected_bytes = 260 * 276 * 2
    actual_bytes   = output_path.stat().st_size
    assert actual_bytes == expected_bytes, \
        f"File size {actual_bytes} != {expected_bytes}"
```

## 7. Diagnostic figure

This is important for catching obvious synthesis failures before feeding the
file into Z01.

```

│  Panel A           │  Panel B             │  Panel C               │
│  2D airglow image  │  1D radial profile   │  Header decode check   │
│  (256×256 px,      │  S(r) vs r (pixels)  │  Re-read row 0,        │
│  imshow, gray)     │  with fringe rings   │  decode key fields,    │
│  Colourbar: ADU    │  labelled            │  print as text         │
└────────────────────┴──────────────────────┴────────────────────────┘
```

**Panel A — 2D image:**
- `imshow(pixel_block[r_start:r_start+256, c_start:c_start+256], cmap='gray', vmin=0, vmax=16383)`
- Title: `"Synthetic airglow image (2D)"`, subtitle showing `t`, `R`, `f`, `α`
- Colourbar labelled `"ADU (14-bit)"`

**Panel B — 1D radial profile:**
- Plot `result['profile_1d']` vs `result['r_grid']` (pixels)
- Mark fringe peak positions with vertical dashed lines
- X-label: `"Radius (pixels)"`, Y-label: `"Intensity (counts)"`

### T6 — Different etalon gaps produce different fringe spacings

> **Implementation deviation (confirmed 2026-04-10):** The as-implemented test
> uses gap values `[10e-3, 15e-3, 20e-3]` m rather than the spec's
> `[19.8e-3, 20.106e-3, 20.4e-3]` m. The narrow operational range caused a
> non-monotonic peak count (14 → 13 → 13) because a boundary fringe at
> t = 19.8 mm falls just inside `r_max` while it falls just outside at
> t = 20.106 mm — a numerical coincidence, not a physics failure. The wider
> range [10, 15, 20] mm produces 7 → 10 → 13 fringes, unambiguously satisfying
> the monotonicity assertion. The physics under test (FSR ∝ 1/t → more fringes
> at larger gap) is identical. **Do not revert to the narrow range.**

```python
def test_fringe_spacing_changes_with_gap():
    """
    A larger etalon gap produces more fringes across r_max.
    FSR ∝ 1/t, so more fringes fit in [0, r_max] at larger t.

    NOTE: gaps use a wide range [10, 15, 20] mm rather than the spec's
    near-operational range [19.8, 20.106, 20.4] mm. The narrow range is
    numerically brittle — a boundary fringe straddles r_max and produces
    a non-monotonic count. See spec T6 deviation note for full explanation.
    """
    from scipy.signal import find_peaks
    gaps = [10e-3, 15e-3, 20e-3]
    n_fringes = []
    for t in gaps:
        _, _, result = synthesise_and_embed(t_m=t, R_refl=0.53, f_lens_m=0.1991)
        profile = result['profile_1d']
        peaks, _ = find_peaks(profile, height=np.mean(profile))
        n_fringes.append(len(peaks))
    assert n_fringes[2] >= n_fringes[1] >= n_fringes[0], \
        f"Fringe count {n_fringes} does not increase with gap"
```

### T7 — Higher reflectivity produces sharper (higher-contrast) fringes

```python
def test_reflectivity_affects_contrast():
    """
    Higher R_refl → higher finesse → higher fringe contrast.
    Contrast = (max - min) / (max + min) of the 1D profile.
    """
    contrasts = {}
    for R in [0.30, 0.53, 0.75]:
        _, _, result = synthesise_and_embed(t_m=20.106e-3, R_refl=R, f_lens_m=0.1991)
        p = result['profile_1d']
        contrasts[R] = (p.max() - p.min()) / (p.max() + p.min())
    assert contrasts[0.75] > contrasts[0.53] > contrasts[0.30], \
        f"Fringe contrast {contrasts} does not increase with reflectivity"
```
