## 1. Purpose and philosophy

INT02 is the second integration script in the WindCube pipeline. It connects
the complete FPI instrument chain in a single run:

```
InstrumentParams (M01)
    ↓
synthesise_calibration_image (M02)   synthesise_airglow_image (M04)
    ↓                                       ↓
make_master_dark / subtract_dark (M03)      │
    ↓                                       │
reduce_calibration_frame (M03)              │
    ↓                                       │
TolanskyPipeline (S13)                      │
    ↓                                       │
fit_calibration_fringe (M05)                │
    ↓                                       ↓
CalibrationResult ──────────── reduce_science_frame (M03)
                                            ↓
                                  fit_airglow_fringe (M06)
                                            ↓
                                  AirglowFitResult
                                            ↓
                              [build_synthetic_metadata (P01)]
```

**What INT02 specifically validates:**

- The Tolansky pipeline correctly characterises `d`, `f`, `α`, `ε`
  from synthetic peak positions (V1)
- M05 recovers the instrument parameters from the synthetic calibration
  fringe to within tolerance (V2, V3)
- `epsilon_cal` is computed correctly from the recovered `t_m` (V4)
- M06 recovers `v_rel` within 20 m/s of the injected truth (noiseless) (V5)
- M06 chi2_reduced is in [0.5, 3.0] on noisy data (V6)
- M06 sigma_v_rel is within the STM wind budget at SNR = 5 (V7)
- `ImageMetadata` correctly carries ground-truth values from the
  synthetic pipeline (V8)

**Philosophy.** INT02 validates the FPI optical chain in isolation — no
geometry pipeline, no orbit propagator. The truth wind speed is injected
directly as `v_rel_ms` rather than derived from NB01/NB02. This isolation
is intentional: if INT02 fails, the problem is in the FPI chain. If INT03
(the full end-to-end script) fails but INT02 passes, the problem is in the
geometry chain or the integration between them.

**Format.** INT02 is a Python script (not a Jupyter notebook). This is
consistent with INT01 and with the repo's tooling. The script lives in
`src/integration/`. It runs all stages sequentially, prints PASS/FAIL for
each verification check, produces diagnostic figures saved to
`outputs/int02_*.png`, and exits with a non-zero return code if any check
fails. It can be run in full or with `--quick` (fewer noise trials, faster).

## 2. Scope decisions

### 2.1 Metadata (P01) is included

`build_synthetic_metadata()` is called to attach an `ImageMetadata` to
each synthetic image. This validates the P01 bridge function and ensures
that `truth_v_los`, `tangent_lat`, `etalon_gap_mm`, and the orbit fields
are populated correctly. V8 checks this explicitly.

The metadata here uses placeholder geometry values (no NB01/NB02 orbit
propagation). `pos_eci_hat`, `vel_eci_hat`, and `attitude_quaternion` are
set to physically plausible constants from S03. Full geometry integration
is deferred to INT03 (S18).

### 2.2 `_build_tolansky_stub` vs `TolanskyPipeline.run()`

For the M05 round-trip test, the `CalibrationResult` fixture is built via
`_build_tolansky_stub(params)` — not by calling `TolanskyPipeline.run()`
on the synthetic fringe profile. This avoids the amplitude-split
reliability problem on synthetic data (see session notes, Fix 5). The
Tolansky pipeline itself is validated separately in Stage B.

### 2.3 Dark frame

A synthetic dark frame is generated using `make_master_dark([dark_array])`
where `dark_array` is a Poisson-sampled dark current image at the nominal
CCD temperature. This exercises the full dark subtraction path through M03
on a physically realistic dark level.

### 2.4 Injection wind speeds for V5 round-trip

Three wind speeds are tested: −300, 0, +300 m/s. This spans the full
science range and checks that the FSR-period scan is working correctly in
both the blueshift and redshift directions.

## 3. Script structure

Ten stages. Each stage is a named function. Stages run sequentially;
a FAIL in any stage does not abort subsequent stages (all stages run and
all failures are reported together at the end).

```
Stage A  — Setup and instrument parameters
Stage B  — Dark frame synthesis and M03 dark subtraction
Stage C  — Calibration image synthesis (M02)
Stage D  — Calibration frame reduction (M03)
Stage E  — Tolansky analysis (S13)  [V1]
Stage F  — Calibration inversion (M05)  [V2, V3, V4]
Stage G  — Airglow image synthesis (M04)
Stage H  — Airglow frame reduction + inversion (M03 + M06)  [V5, V6, V7]
Stage I  — Metadata validation (P01)  [V8]
Stage J  — Summary report and figure export
```

## 6. Stage B — Dark frame synthesis and subtraction

```python
def stage_b_dark(config: dict) -> dict:
    """
    Synthesise a dark frame at the nominal CCD operating temperature.
    Build master dark. Verify dark level is physically reasonable.
    Return dark arrays for use in Stages C and G.
    """
```

Implementation:

1. Generate a dark frame as Poisson-sampled dark current:
   ```python
   from src.constants import CCD_DARK_RATE_E_PX_S, CCD_TEMP_NOMINAL_C
   # Nominal dark rate at operating temperature
   dark_rate_adu_per_s = CCD_DARK_RATE_E_PX_S   # from S03
   exp_time_s = config["params"].exp_time_s      # if defined; else use 5.0 s
   dark_level = dark_rate_adu_per_s * exp_time_s
   dark_array = config["rng_dark"].poisson(
       dark_level,
       size=(int(config["params"].r_max * 2), int(config["params"].r_max * 2))
   ).astype(np.uint16)
   ```

   **Implementation note:** `InstrumentParams` may not define `exp_time_s`.
   If it does not, use `exp_time_s = 5.0` (a typical calibration exposure
   at 500 cs). Check the actual `InstrumentParams` attribute before using.

2. `master_dark = make_master_dark([dark_array])`

3. Print:
   ```
   [Stage B] Dark frame:
     Expected dark level : {dark_level:.1f} ADU (Poisson mean)
     Actual dark mean    : {dark_array.mean():.1f} ± {dark_array.std():.1f} ADU
     Master dark mean    : {master_dark.mean():.1f} ADU
   ```

4. Return `{"dark_array": dark_array, "master_dark": master_dark}`

def main():
    parser = argparse.ArgumentParser(description="INT02 FPI chain integration")

                        help="Reduce noise trials for faster run")
