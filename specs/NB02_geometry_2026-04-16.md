## 1. Purpose and overview

NB02 bridges the orbit propagator (NB01) and the FPI fringe forward model.
For every epoch in the NB01 output it computes:

1. The instrument LOS unit vector in ECI J2000 — from spacecraft attitude
   quaternion and instrument boresight geometry.
2. The tangent point geodetic coordinates — the point on the 250 km
   emission altitude shell that the LOS intersects.
3. The relative LOS wind speed `v_rel` — the Doppler-shift observable —
   combining atmospheric wind, spacecraft velocity, and Earth rotation,
   all projected onto the LOS.
4. The L1c inverse — removing spacecraft and Earth-rotation terms from a
   recovered `v_rel` to yield the physical atmospheric wind.

**Why this is the most mathematically critical module in the pipeline.**
The spacecraft moves at ~7,600 m/s. The atmospheric wind is ~10–400 m/s.
The LOS projection of spacecraft velocity dominates the Doppler shift by
a factor of 20–750. A 0.1° boresight error → `7600 × sin(0.1°) ≈ 13 m/s`
wind bias — larger than the 9.8 m/s STM precision requirement. Every
calculation in this module must be exact.

NB02 is implemented as four independent, sequentially testable sub-modules:

| File | Sub-problem | Input → Output |
|------|-------------|----------------|
| `nb02a_boresight_2026-04-16.py` | Quaternion → LOS unit vector | attitude q → `los_eci` |
| `nb02b_tangent_point_2026-04-16.py` | Ray–ellipsoid intersection | `los_eci` → tangent point geodetic coords |
| `nb02c_los_projection_2026-04-16.py` | Wind + spacecraft → `v_rel` | `WindMap`, epoch, geometry → `v_rel` |
| `nb02d_l1c_calibrator_2026-04-16.py` | `v_rel` → `v_wind_LOS` | remove V_sc and v_earth contributions |

## 2. Coordinate frames and conventions

NB02 works in four frames. The authoritative definitions for the spacecraft
frames are in the AOCS Design Report (SI-UCAR-WC-RP-004 §2.4.2) and are
summarised in NB01 spec §2.4. The descriptions below must remain consistent
with those sources.

### 2.1 Satellite Body Reference Frame (BRF)

The BRF is fixed to the spacecraft structure. Axis definitions from
SI-UCAR-WC-RP-004 §2.4.2.1:

| Axis | Direction | Physical meaning |
|------|-----------|-----------------|
| **−X** (BRF) | Payload boresight | The FPI optical axis points in the **−X** direction. The instrument aperture faces **−X**. |
| **+Y** (BRF) | Star Tracker boresight | The Jena Optronik ASTRO-CL FOV points in **+Y** |
| **+Z** (BRF) | Completes right-handed basis | Approximately toward Sun in along-track fine pointing |

**Critical distinction from earlier drafts:** the payload boresight is
`−X_BRF`, not `+X_BRF`. All boresight vectors constructed in NB02a must
point in the **−X** body-frame direction before being rotated to ECI.
Any code that constructs `boresight_body = np.array([1., 0., 0.])` is
wrong; the correct form is `boresight_body = np.array([-1., 0., 0.])`.

For the two THRF observation configurations (SI-UCAR-WC-RP-004 §2.4.2.2):

- **Along-track:** `−X_BRF` is depressed 15.73° below local horizontal
  and points forward in the orbit-velocity direction. The tangent point
  is ~923 km **ahead** of the spacecraft.
- **Cross-track:** `−X_BRF` is depressed 15.73° below local horizontal
  and points perpendicular to the orbit plane. The tangent point is
  ~923 km to the **side** of the spacecraft ground track.

Variable name convention: `boresight_body` for the 3-vector in BRF;
`los_eci` for the same vector after rotation to ECI J2000.

### 2.2 ECI J2000

Earth-Centred Inertial, non-rotating, fixed to distant stars.
Origin at Earth's centre. All orbit propagation (from NB01) and attitude
quaternions are expressed in this frame. Epoch-independent for time spans
< 1 day. All `pos_eci_*` and `vel_eci_*` columns from NB01 are in this frame.

Variable name convention: vectors in ECI carry the `_eci` suffix.

### 2.3 ECEF (ITRS)

Earth-Centred Earth-Fixed, rotates with Earth. Used for geodetic coordinate
transforms only. Never stored; always converted immediately via astropy
GCRS→ITRS. Not to be confused with BRF or THRF.

### 2.4 ENU at the tangent point

East-North-Up frame centred at the tangent point geodetic location.
`v_zonal` is the East component; `v_merid` is the North component.
Used to express atmospheric wind vectors sampled from the WindMap (NB00).

### 2.5 Quaternion convention

Scalar-last `[q1, q2, q3, q4]` where q4 is the scalar part. This is the
Space Inventor ADCS convention used throughout the WindCube AOCS system
(SI-UCAR-WC-RP-004). Never assume scalar-first without verifying the source.

The attitude quaternion `q` represents the rotation from BRF to ECI J2000:
a vector `v_brf` in body frame transforms to ECI as
`v_eci = R(q) @ v_brf` where `R(q)` is the rotation matrix derived from `q`.

### 2.6 THRF and SIRF — pipeline boundary

The Tangent Height Reference Frame (THRF) and Star Tracker Interface
Reference Frame (SIRF) are defined in SI-UCAR-WC-RP-004 §2.4.2.2 and
§2.4.2.3 respectively and summarised in NB01 §2.4.2 and §2.4.3.

NB02 does **not** implement THRF or SIRF transforms directly. The AOCS
onboard software closes the THRF→SIRF→PIRF→PSRF chain in real time.
NB02's role in the frame hierarchy is:

```
ECI J2000  →  THRF geometry  →  [AOCS onboard]  →  SIRF → PIRF → PSRF
              ^^^^^^^^^^^^^^
              NB02 computes boresight, tangent point,
              and LOS projections in ECI J2000.
              BRF→ECI rotation uses the attitude quaternion
              reported in spacecraft telemetry (SIRF-derived).
```

AOCS performance requirements (APE < 100 arcsec, AKE < 30 arcsec,
RPE < 72 arcsec) are expressed in SIRF, not in NB02's output. The 0.1°
boresight error budget noted in §1 above encompasses the full THRF→PSRF
chain and must not be confused with per-frame AOCS residuals.

## 3. Physical geometry constants

All values from S03. Listed here for quick reference:

| Constant | Value | Description |
|----------|-------|-------------|
| `SC_ALTITUDE_KM` | 510.0 km | Spacecraft geodetic altitude (confirmed mission value) |
| `TP_ALTITUDE_KM` | 250.0 km | OI 630 nm tangent point target altitude |
| `DEPRESSION_ANGLE_DEG` | 15.73° | arccos(6621/6881) — see derivation below |
| `TP_LEAD_KM` | ~923 km | Horizontal distance tangent point leads spacecraft |
| `TP_LEAD_S` | ~122 s | Flight time ahead at 7597 m/s orbital speed |
| `WGS84_A_M` | 6,378,137.0 m | WGS84 equatorial radius |
| `WGS84_B_M` | 6,356,752.3 m | WGS84 polar radius |
| `EARTH_OMEGA_RAD_S` | 7.2921150e-5 rad/s | Earth rotation rate |

**Depression angle derivation (arccos, not arctan):**
```
R_E  = 6,371,000 m  (mean radius — adequate for angle geometry)
R_sc = R_E + 510,000 = 6,881,000 m
R_tp = R_E + 250,000 = 6,621,000 m

In the right triangle (right angle at tangent point T):
  cos(δ) = R_tp / R_sc = 6,621,000 / 6,881,000 = 0.96222
  δ = arccos(0.96222) = 15.73°

Tangent point leads spacecraft by:
  d = sqrt(R_sc² − R_tp²) ≈ (R_sc − R_tp) / tan(δ)
    = 260,000 / tan(15.73°) ≈ 923 km
```

**BORESIGHT DIRECTION — critical geometry note:**
The instrument boresight is the **−X_BRF** axis (SI-UCAR-WC-RP-004 §2.4.2.1).
For along-track observations, the boresight points forward in the velocity
direction (THRF along-track configuration) and is depressed 15.73° toward
the limb. The tangent point is therefore **ahead** of the spacecraft by
~923 km. WindCube measures the thermosphere it will fly over approximately
122 seconds in the future.

For cross-track observations, the boresight points perpendicular to the
orbit plane (THRF cross-track configuration), also depressed 15.73°.

### 4.1 NB02a — Boresight

```python
# geometry/nb02a_boresight_2026-04-16.py

DEPRESSION_ANGLE_DEG = 15.73   # arccos(6621/6881), 510 km alt, 250 km tangent
DEFAULT_ALTITUDE_KM  = 510.0   # confirmed mission altitude

# ---------------------------------------------------------------------------
# BORESIGHT DIRECTION NOTE (SI-UCAR-WC-RP-004 §2.4.2.1)
# The FPI payload boresight is the -X_BRF axis. The body-frame boresight
# vector is always np.array([-1., 0., 0.]). Rotating this by the attitude
# quaternion q (BRF→ECI) gives los_eci pointing toward the tangent point.
# For along_track mode: los_eci ≈ forward velocity direction, depressed 15.73°.
# For cross_track mode: los_eci ≈ orbit-normal direction, depressed 15.73°.
# These correspond to the THRF along-track and cross-track configurations
# defined in SI-UCAR-WC-RP-004 §2.4.2.2. See NB01 spec §2.4.2.
# ---------------------------------------------------------------------------

def _compute_depression_angle(
    altitude_km: float,
    h_target_km: float,
) -> float:
    """
    Depression angle below local horizontal for limb observation.
    Uses right-triangle geometry (right angle at tangent point).
    depression = arccos((R_E + h_target) / (R_E + altitude))
    R_E = 6371.0 km (mean radius — adequate for angle geometry).
    """

def compute_synthetic_quaternion(
    pos_eci: np.ndarray,    # shape (3,), ECI position, m
    vel_eci: np.ndarray,    # shape (3,), ECI velocity, m/s
    look_mode: str,         # 'along_track' or 'cross_track'
    altitude_km: float = DEFAULT_ALTITUDE_KM,
    h_target_km: float = 250.0,
) -> np.ndarray:
    """
    Synthesise a unit attitude quaternion for a given look mode.

    The FPI boresight is the -X_BRF axis (SI-UCAR-WC-RP-004 §2.4.2.1).
    This function constructs the quaternion q rotating BRF→ECI such that
    R(q) @ [-1, 0, 0] points toward the tangent point.

    For along_track (THRF along-track configuration):
        -X_BRF rotated to ECI points forward in velocity direction,
        depressed 15.73° below local horizontal.
        Tangent point is ~923 km ahead of spacecraft.

    For cross_track (THRF cross-track configuration):
        -X_BRF rotated to ECI points perpendicular to orbit plane,
        depressed 15.73° below local horizontal.
        Tangent point is ~923 km to the side of the ground track.

    The look_mode strings 'along_track' and 'cross_track' match the
    THRF configuration names in SI-UCAR-WC-RP-004 §2.4.2.2 and the
    thrf_config convention established in NB01 spec §2.4.2.

    Returns
    -------
    q : np.ndarray, shape (4,)
        Scalar-last unit quaternion [q1, q2, q3, q4] rotating BRF to ECI.
    """

def compute_los_eci(
    pos_eci: np.ndarray,    # shape (3,), m
    vel_eci: np.ndarray,    # shape (3,), m/s
    look_mode: str,         # 'along_track' or 'cross_track'
    altitude_km: float = DEFAULT_ALTITUDE_KM,
    h_target_km: float = 250.0,
) -> tuple[np.ndarray, np.ndarray]:
    """
    Compute the LOS unit vector in ECI and the corresponding quaternion.

    Internally builds boresight_body = np.array([-1., 0., 0.]) (-X_BRF),
    then rotates to ECI using the attitude quaternion for the given look_mode.

    Returns
    -------
    (los_eci, q) :
        los_eci : np.ndarray, shape (3,) — unit LOS vector in ECI J2000
        q       : np.ndarray, shape (4,) — attitude quaternion used (BRF→ECI)
    """
```

```python
# geometry/nb02b_tangent_point_2026-04-16.py

def compute_tangent_point(
    pos_eci: np.ndarray,    # shape (3,), spacecraft ECI position, m
    los_eci: np.ndarray,    # shape (3,), unit LOS vector in ECI
    epoch,                  # astropy Time object
    h_target_km: float = 250.0,
) -> dict:
    """
    Find the tangent point via ray-WGS84 ellipsoid intersection at the
    target emission altitude shell (ellipsoid inflated by h_target).

    Uses the smaller positive root (near-side intersection). If the ray
    does not intersect the inflated ellipsoid, raises ValueError.

    The los_eci input is produced by NB02a compute_los_eci() and points
    in the -X_BRF direction rotated to ECI. For along_track mode the
    tangent point will be ~923 km ahead of the spacecraft in the velocity
    direction (THRF along-track configuration). For cross_track mode it
    will be ~923 km to the side (THRF cross-track configuration).

    The AOCS THRF model error requirement is < 5 km from the true 250 km
    tangent height (SI-UCAR-WC-RP-004 §2.1). The WGS84 oblateness causes
    tp_alt_km to differ from h_target_km by up to ~5 km; this is expected
    and verified by test T5.

    Returns
    -------
    dict with keys:
        'tp_eci'     : np.ndarray, shape (3,) — tangent point ECI position, m
        'tp_lat_deg' : float — geodetic latitude, degrees
        'tp_lon_deg' : float — geodetic longitude, degrees (-180 to +180)
        'tp_alt_km'  : float — geodetic altitude, km
                       (will differ slightly from h_target_km due to WGS84
                       oblateness; expected error < 5 km — see T5)
    """
```

### 4.3 NB02c — LOS Projection

```python
# geometry/nb02c_los_projection_2026-04-16.py

def enu_unit_vectors_eci(
    lat_deg: float,
    lon_deg: float,
    epoch,              # astropy Time
) -> tuple[np.ndarray, np.ndarray, np.ndarray]:
    """
    Compute East, North, Up unit vectors in ECI at a geodetic location.

    Uses astropy ITRS→GCRS for the ECEF→ECI rotation.
    This transform is epoch-dependent (precession + nutation).
    Never cache across epochs.

    Returns (e_east_eci, e_north_eci, e_up_eci), each shape (3,).
    """

def earth_rotation_velocity_eci(
    tp_eci: np.ndarray,     # tangent point ECI position, m
) -> np.ndarray:
    """
    Velocity of the atmosphere at the tangent point due to Earth rotation.
    v_earth = omega_E × r_tp  (cross product)
    omega_E = [0, 0, EARTH_OMEGA_RAD_S] in ECI (aligned with rotation axis)

    Returns v_earth_eci : shape (3,), m/s
    """

def compute_v_rel(
    wind_map,               # WindMap instance (any NB00 subclass)
    tp_lat_deg: float,
    tp_lon_deg: float,
    tp_eci: np.ndarray,     # shape (3,), m
    vel_eci: np.ndarray,    # spacecraft ECI velocity, shape (3,), m/s
    los_eci: np.ndarray,    # unit LOS vector in ECI, shape (3,)
    epoch,                  # astropy Time
) -> dict:
    """
    Compute v_rel — the FPI's measured Doppler shift observable.

    v_rel = v_wind_LOS - V_sc_LOS - v_earth_LOS

    where all three terms are LOS projections:
      v_wind_LOS  = (R_enu2eci @ [v_zonal, v_merid, 0]) · los_eci
      V_sc_LOS    = vel_eci · los_eci
      v_earth_LOS = (omega_E × tp_eci) · los_eci

    Sign convention: positive v_rel = emitting gas receding from instrument
    (redshift), corresponding to LOS wind away from the spacecraft.

    The los_eci argument is the -X_BRF boresight rotated to ECI (from NB02a).
    The vel_eci argument is from NB01 output columns vel_eci_x/y/z (ECI J2000).

    Parameters
    ----------
    wind_map    : WindMap — called as wind_map.sample(tp_lat_deg, tp_lon_deg)
                  Returns (v_zonal_ms, v_merid_ms).

    Returns
    -------
    dict with keys:
        'v_rel'        : float, m/s — full observable (what FPI measures)
        'v_wind_LOS'   : float, m/s — atmospheric wind LOS component
        'V_sc_LOS'     : float, m/s — spacecraft velocity LOS component
        'v_earth_LOS'  : float, m/s — Earth rotation LOS component
        'v_zonal_ms'   : float, m/s — zonal wind at tangent point (from WindMap)
        'v_merid_ms'   : float, m/s — meridional wind (from WindMap)
    """
```

### 4.4 NB02d — L1c Calibrator

```python
# geometry/nb02d_l1c_calibrator_2026-04-16.py

def compose_v_rel(
    v_wind_LOS: float,
    V_sc_LOS: float,
    v_earth_LOS: float,
) -> float:
    """
    Forward composition: combine the three LOS velocity terms into v_rel.
    v_rel = v_wind_LOS - V_sc_LOS - v_earth_LOS
    Used in the synthetic forward pipeline to build the ground truth.

    This is the forward model used in the synthetic image generator (M04)
    to embed a known wind signal. The inverse is remove_spacecraft_velocity().
    """

def remove_spacecraft_velocity(
    v_rel: float,
    V_sc_LOS: float,
    v_earth_LOS: float,
) -> float:
    """
    Inverse (L1c step): recover v_wind_LOS from the measured v_rel.
    v_wind_LOS = v_rel + V_sc_LOS + v_earth_LOS

    Applied to real or simulated FPI measurements after M06 has recovered
    v_rel from the fringe pattern. The recovered v_wind_LOS is then passed
    to M07 for WLS wind vector retrieval.

    Round-trip accuracy requirement: < 1e-10 m/s (machine precision).
    If this test fails, there is a sign error — not a numerical issue.
    """
```

```python
def test_los_depression_angle():
    """
    LOS must be depressed ~15.73° below local horizontal toward Earth.
    Expected: 10° < depression < 22° (allows for orbit geometry variation).
    """
    pos = np.array([0., 6.896e6, 0.])
```

### T6 — Tangent point leads spacecraft (−X_BRF geometry)
```python
def test_tangent_point_leads_spacecraft():
    """
    Along-track tangent point must be AHEAD of spacecraft, not behind.
    The along-track offset must be > 500 km forward.

    Geometry: -X_BRF boresight points forward (velocity direction) in
    along_track THRF configuration, depressed 15.73°. The tangent point
    is ~923 km ahead (SI-UCAR-WC-RP-004 §2.4.2.2).
    """
    from astropy.time import Time
    pos = np.array([0., 6.896e6, 0.])   # at equator, on Y-axis
    vel = np.array([977.2, 0., 7534.5]) # sun-synchronous direction
    epoch = Time('2027-01-01T00:00:00', scale='utc')
    los, _ = compute_los_eci(pos, vel, 'along_track')
    tp = compute_tangent_point(pos, los, epoch)
    tp_pos = tp['tp_eci']
    # Project (tp - sc) onto velocity direction to get forward offset
    v_hat = vel / np.linalg.norm(vel)
    forward_offset_m = np.dot(tp_pos - pos, v_hat)
    assert forward_offset_m > 500e3, \
        f"Tangent point {forward_offset_m/1e3:.0f} km from sc; expected > 500 km FORWARD"
```

### T7 — v_rel round-trip < 1e-10 m/s
```python
def test_v_rel_round_trip():
    """
    compose_v_rel / remove_spacecraft_velocity must be exact inverses.
    Error must be < 1e-10 m/s (machine precision).
    If this fails, there is a sign error — not a numerical issue.
    """
    for v_wind, V_sc, v_earth in [
        (100.0, -7100.0, 460.0),
        (-50.0,  -200.0, 230.0),
        (  0.0,  7097.3, 461.1),
    ]:
        v_rel = compose_v_rel(v_wind, V_sc, v_earth)
        v_wind_recovered = remove_spacecraft_velocity(v_rel, V_sc, v_earth)
        assert abs(v_wind_recovered - v_wind) < 1e-10, \
            f"Round-trip error {abs(v_wind_recovered - v_wind):.2e} m/s — check signs"
```

### T8 — Along-track V_sc_LOS >> cross-track V_sc_LOS
```python
def test_vsc_los_ratio():
    """
    Along-track V_sc_LOS must be > 10× cross-track V_sc_LOS.
    Verifies that the look modes are correctly implemented.
    In along_track mode the -X_BRF boresight is nearly aligned with
    the velocity vector, maximising the spacecraft Doppler contribution.
    """
    from astropy.time import Time
    from windmap.nb00_wind_map import UniformWindMap
    pos = np.array([0., 6.896e6, 0.])
    vel = np.array([977.2, 0., 7534.5])
    epoch = Time('2027-01-01T00:00:00', scale='utc')
    wind_map = UniformWindMap(v_zonal_ms=0.0, v_merid_ms=0.0)

    results = {}
    for mode in ['along_track', 'cross_track']:
        los, _ = compute_los_eci(pos, vel, mode)
        tp = compute_tangent_point(pos, los, epoch)
        res = compute_v_rel(wind_map,
                            tp['tp_lat_deg'], tp['tp_lon_deg'], tp['tp_eci'],
                            vel, los, epoch)
        results[mode] = abs(res['V_sc_LOS'])

    ratio = results['along_track'] / (results['cross_track'] + 1e-6)
    assert ratio > 10.0, \
        f"V_sc_LOS ratio along/cross = {ratio:.1f}; expected > 10"
```

```python
    from astropy.time import Time

        assert abs(np.linalg.norm(n) - 1.0) < 1e-10
```

### Reference frame check before writing nb02a

The single most common error in NB02a is using `+X_BRF` as the boresight
direction. **The correct boresight is `−X_BRF`.** Before writing any code,
confirm you understand:

- `boresight_body = np.array([-1., 0., 0.])` ← correct
- `boresight_body = np.array([1., 0., 0.])` ← wrong; do not use

The attitude quaternion `q` rotates BRF→ECI (scalar-last convention).
`los_eci = R(q) @ boresight_body` where `R(q)` is the rotation matrix.
