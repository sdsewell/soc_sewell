## 1. Purpose and overview

NB02 bridges the orbit propagator (NB01) and the FPI fringe forward model.
For every epoch in the NB01 output it computes:

1. The instrument LOS unit vector in ECI J2000 — from spacecraft attitude
   quaternion and instrument boresight geometry.
2. The tangent point geodetic coordinates — the point on the 250 km
   emission altitude shell that the LOS intersects.
3. The relative LOS wind speed `v_rel` — the Doppler-shift observable —
   combining atmospheric wind, spacecraft velocity, and Earth rotation,
   all projected onto the LOS.
4. The L1c inverse — removing spacecraft and Earth-rotation terms from a
   recovered `v_rel` to yield the physical atmospheric wind.

**Why this is the most mathematically critical module in the pipeline.**
The spacecraft moves at ~7,600 m/s. The atmospheric wind is ~10–400 m/s.
The LOS projection of spacecraft velocity dominates the Doppler shift by
a factor of 20–750. A 0.1° boresight error → `7600 × sin(0.1°) ≈ 13 m/s`
wind bias — larger than the 9.8 m/s STM precision requirement. Every
calculation in this module must be exact.

NB02 is implemented as four independent, sequentially testable sub-modules:

| File | Sub-problem | Input → Output |
|------|-------------|----------------|
| `nb02a_boresight_2026_04_16.py` | Quaternion → LOS unit vector | attitude q → `los_eci` |
| `nb02b_tangent_point_2026_04_16.py` | Ray–ellipsoid intersection | `los_eci` → tangent point geodetic coords |
| `nb02c_los_projection_2026_04_16.py` | Wind + spacecraft → `v_rel` | `WindMap`, epoch, geometry → `v_rel` |
| `nb02d_l1c_calibrator_2026_04_16.py` | `v_rel` → `v_wind_LOS` | remove V_sc and v_earth contributions |

## 2. Coordinate frames and conventions

NB02 works in four frames. The authoritative definitions for the spacecraft
frames are in the AOCS Design Report (SI-UCAR-WC-RP-004 §2.4.2) and are
summarised in NB01 spec §2.4. The descriptions below must remain consistent
with those sources.

### 2.1 Satellite Body Reference Frame (BRF)

The BRF is fixed to the spacecraft structure. Axis definitions from
SI-UCAR-WC-RP-004 §2.4.2.1:

| Axis | Direction | Physical meaning |
|------|-----------|-----------------|
| **−X** (BRF) | Payload boresight | The FPI optical axis points in the **−X** direction. The instrument aperture faces **−X**. |
| **+Y** (BRF) | Star Tracker boresight | The Jena Optronik ASTRO-CL FOV points in **+Y** |
| **+Z** (BRF) | Completes right-handed basis | Approximately toward Sun in along-track fine pointing |

**Critical distinction from earlier drafts:** the payload boresight is
`−X_BRF`, not `+X_BRF`. All boresight vectors constructed in NB02a must
point in the **−X** body-frame direction before being rotated to ECI.
Any code that constructs `boresight_body = np.array([1., 0., 0.])` is
wrong; the correct form is `boresight_body = np.array([-1., 0., 0.])`.

For the two THRF observation configurations (SI-UCAR-WC-RP-004 §2.4.2.2):

- **Along-track:** `−X_BRF` is depressed 15.73° (nominal equatorial value)
  below local horizontal and points forward in the orbit-velocity direction.
  The tangent point is ~923 km **ahead** of the spacecraft.
- **Cross-track:** `−X_BRF` is depressed 15.73° below local horizontal
  and points perpendicular to the orbit plane. The tangent point is ~923 km
  to the **side** of the spacecraft ground track.

Variable name convention: `boresight_body` for the 3-vector in BRF;
`los_eci` for the same vector after rotation to ECI J2000.

### 2.2 ECI J2000

Earth-Centred Inertial, non-rotating, fixed to distant stars.
All orbit propagation (from NB01) and attitude quaternions are expressed in
this frame. All `pos_eci_*` and `vel_eci_*` columns from NB01 are in this frame.

Variable name convention: vectors in ECI carry the `_eci` suffix.

### 2.3 ECEF (ITRS)

Earth-Centred Earth-Fixed, rotates with Earth. Used for geodetic coordinate
transforms only. Never stored; always converted immediately via astropy
GCRS→ITRS.

### 2.4 ENU at the tangent point

East-North-Up frame centred at the tangent point geodetic location.
`v_zonal` is the East component; `v_merid` is the North component.
Used to express atmospheric wind vectors sampled from the WindMap (NB00).

### 2.5 Quaternion convention

Scalar-last `[q1, q2, q3, q4]` where q4 is the scalar part. This is the
Space Inventor ADCS convention used throughout the WindCube AOCS system
(SI-UCAR-WC-RP-004).

The attitude quaternion `q` represents the rotation from BRF to ECI J2000:
`v_eci = R(q) @ v_brf` where `R(q)` is the rotation matrix derived from `q`.

### 2.6 THRF and SIRF — pipeline boundary

NB02 does **not** implement THRF or SIRF transforms directly. NB02's role:

```
ECI J2000  →  THRF geometry  →  [AOCS onboard]  →  SIRF → PIRF → PSRF
              ^^^^^^^^^^^^^^
              NB02 computes boresight, tangent point,
              and LOS projections in ECI J2000.
```

## 3. Physical geometry constants

All values from S03:

| Constant | Value | Description |
|----------|-------|-------------|
| `SC_ALTITUDE_KM` | 510.0 km | Spacecraft geodetic altitude |
| `TP_ALTITUDE_KM` | 250.0 km | OI 630 nm tangent point target altitude |
| `DEPRESSION_ANGLE_DEG` | 15.73° | Equatorial approximation — used as **iteration seed only** (see §4.1) |
| `TP_LEAD_KM` | ~923 km | Horizontal distance tangent point leads spacecraft |
| `TP_LEAD_S` | ~122 s | Flight time ahead at 7597 m/s orbital speed |
| `WGS84_A_M` | 6,378,137.0 m | WGS84 equatorial radius |
| `WGS84_B_M` | 6,356,752.3 m | WGS84 polar radius |
| `EARTH_OMEGA_RAD_S` | 7.2921150e-5 rad/s | Earth rotation rate |

**Why `DEPRESSION_ANGLE_DEG` is a seed, not a final value:**

The depression angle δ = arccos(R_tp / R_sc) depends on `R_tp`, the
distance from Earth's centre to the tangent point on the emission altitude
shell. Because WGS84 is an oblate spheroid, `R_tp` varies with geocentric
latitude:

```
R_tp(φ_geoc) = R_WGS84(φ_geoc) + h_target
```

For a 97.44° SSO the spacecraft and its along-track tangent point can differ
by up to ~30° in geocentric latitude. The WGS84 radius difference from
equator to pole is ~21 km. Plugging the spacecraft latitude into R_tp
instead of the tangent-point latitude therefore produces depression-angle
errors that shift `tp_alt` by up to ~6 km, exceeding the ±5 km V1
tolerance in INT01.

The 15.73° equatorial value shown above (arccos(6621/6881)) is retained as a
named constant and used as the iteration seed. The converged value returned
by `_compute_depression_angle()` will differ by up to ~0.1° from 15.73°
at high latitudes — this is expected and correct.

## 4. Function signatures

### 4.1 NB02a — Boresight

```python
# geometry/nb02a_boresight_2026_04_16.py

DEPRESSION_ANGLE_DEG = 15.73   # equatorial approximation; iteration seed only
DEFAULT_ALTITUDE_KM  = 510.0
DEFAULT_H_TARGET_KM  = 250.0

# ---------------------------------------------------------------------------
# BORESIGHT DIRECTION NOTE (SI-UCAR-WC-RP-004 §2.4.2.1)
# The FPI payload boresight is the -X_BRF axis. The body-frame boresight
# vector is always np.array([-1., 0., 0.]). Rotating this by the attitude
# quaternion q (BRF→ECI) gives los_eci pointing toward the tangent point.
# For along_track mode: los_eci ≈ forward velocity direction, depressed ~15.73°.
# For cross_track mode: los_eci ≈ orbit-normal direction, depressed ~15.73°.
# These correspond to the THRF along-track and cross-track configurations
# defined in SI-UCAR-WC-RP-004 §2.4.2.2. See NB01 spec §2.4.2.
# ---------------------------------------------------------------------------

def _wgs84_geocentric_radius(lat_geoc_rad: float) -> float:
    """
    WGS84 geocentric radius at the given geocentric latitude.

    R(φ) = sqrt( (a² cos φ)² + (b² sin φ)² ) / sqrt( (a cos φ)² + (b sin φ)² )

    where a = WGS84_A_M, b = WGS84_B_M.
    Returns radius in metres.
    """

def _compute_depression_angle(
    pos_eci: np.ndarray,
    vel_eci: np.ndarray,
    look_mode: str,
    h_target_km: float = DEFAULT_H_TARGET_KM,
    n_iter: int = 5,
) -> float:
    """
    Depression angle (radians) below local horizontal for a limb observation
    targeting h_target_km tangent height, accounting for WGS84 oblateness.

    The naive formula δ = arccos(R_tp / R_sc) with a fixed mean-sphere R_tp
    produces tp_alt errors up to ~6 km on a 97.44° SSO because the tangent
    point can be ~30° different in geocentric latitude from the spacecraft,
    where the WGS84 radius differs by up to ~21 km. This function converges
    to the correct value using 5 iterations.

    Algorithm
    ---------
    1.  Compute R_sc = |pos_eci|.
    2.  Estimate initial R_tp using R_WGS84 at the spacecraft geocentric
        latitude plus h_target_km.
    3.  δ = arccos(R_tp / R_sc).
    4.  Project the boresight direction (approximate) forward by δ from the
        spacecraft to find an approximate TP ECI position.
    5.  Compute the TP geocentric latitude from the approximate TP position.
    6.  Recompute R_tp = R_WGS84(φ_tp_geoc) + h_target_km.
    7.  Repeat from step 3 for n_iter iterations.

    After 5 iterations the tp_alt error is < 0.1 km across the full polar
    orbit (verified by INT01, 16 orbits, 8641 epochs).

    Parameters
    ----------
    pos_eci    : np.ndarray, shape (3,) — spacecraft ECI position, metres.
    vel_eci    : np.ndarray, shape (3,) — spacecraft ECI velocity, m/s.
    look_mode  : str — 'along_track' or 'cross_track'. Used to determine
                 the approximate boresight direction for TP projection.
    h_target_km: float — target tangent height, km. Default 250.0.
    n_iter     : int — number of iterations. Default 5; do not reduce below 3.

    Returns
    -------
    float : converged depression angle, radians.

    Notes
    -----
    - The module-level constant DEPRESSION_ANGLE_DEG (≈ 15.73°) is kept for
      documentation and as the seed for the first iteration. Do not use it
      as the final depression angle.
    - Callers (compute_synthetic_quaternion, compute_los_eci) must pass both
      pos_eci and vel_eci — the function signature changed from the earlier
      scalar (altitude_km, h_target_km) form.
    """

def compute_synthetic_quaternion(
    pos_eci: np.ndarray,    # shape (3,), ECI position, m
    vel_eci: np.ndarray,    # shape (3,), ECI velocity, m/s
    look_mode: str,         # 'along_track' or 'cross_track'
    h_target_km: float = DEFAULT_H_TARGET_KM,
) -> np.ndarray:
    """
    Synthesise a unit attitude quaternion for a given look mode.

    The FPI boresight is the -X_BRF axis (SI-UCAR-WC-RP-004 §2.4.2.1).
    Constructs q rotating BRF→ECI such that R(q) @ [-1, 0, 0] points toward
    the tangent point at h_target_km.

    Internally calls _compute_depression_angle(pos_eci, vel_eci, look_mode,
    h_target_km) to get the orbit-position-dependent depression angle before
    constructing the rotation.

    Returns
    -------
    q : np.ndarray, shape (4,) — scalar-last unit quaternion [q1, q2, q3, q4].
    """

def compute_los_eci(
    pos_eci: np.ndarray,    # shape (3,), m
    vel_eci: np.ndarray,    # shape (3,), m/s
    look_mode: str,         # 'along_track' or 'cross_track'
    h_target_km: float = DEFAULT_H_TARGET_KM,
) -> tuple[np.ndarray, np.ndarray]:
    """
    Compute the LOS unit vector in ECI and the corresponding attitude quaternion.

    Internally:
    1. Calls _compute_depression_angle(pos_eci, vel_eci, look_mode, h_target_km)
       to obtain the orbit-position-dependent depression angle.
    2. Constructs boresight_body = np.array([-1., 0., 0.]) (-X_BRF).
    3. Rotates to ECI using the attitude quaternion for look_mode.

    Returns
    -------
    (los_eci, q) :
        los_eci : np.ndarray, shape (3,) — unit LOS vector in ECI J2000
        q       : np.ndarray, shape (4,) — attitude quaternion used (BRF→ECI,
                  scalar-last)
    """
```

```python
def compute_tangent_point(
    pos_eci: np.ndarray,    # shape (3,), spacecraft ECI position, m
    los_eci: np.ndarray,    # shape (3,), unit LOS vector in ECI
    epoch,                  # astropy Time object
    h_target_km: float = 250.0,
) -> dict:
    """
    Find the tangent point via ray-WGS84 ellipsoid intersection at the
    target emission altitude shell (ellipsoid inflated by h_target).

    Uses the smaller positive root (near-side intersection). If the ray
    does not intersect the inflated ellipsoid, raises ValueError.

    The los_eci input is produced by NB02a compute_los_eci() and points
    in the -X_BRF direction rotated to ECI. For along_track mode the
    tangent point is ~923 km ahead of the spacecraft; for cross_track ~923 km

    near-tangencies without false negatives.

    -------
    dict with keys:
        'tp_eci'     : np.ndarray, shape (3,) — tangent point ECI position, m
        'tp_lat_deg' : float — geodetic latitude, degrees
        'tp_lon_deg' : float — geodetic longitude, degrees (-180 to +180)
        'tp_alt_km'  : float — geodetic altitude, km.
                       With the iterative depression angle from NB02a, this
                       deviates from h_target_km by < 0.1 km in practice.
                       The ±5 km tolerance is the physical requirement.
    """
```

### T4 — LOS depression angle ≈ 15.73°

```python
def test_los_depression_angle():
    """
    LOS must be depressed ~15.73° below local horizontal toward Earth.
    Window [10°, 22°] allows for iterative convergence variation.
    """
    pos = np.array([0., 6.896e6, 0.])
    vel = np.array([977.2, 0., 7534.5])
    nadir = -pos / np.linalg.norm(pos)
    for mode in ['along_track', 'cross_track']:
        los, _ = compute_los_eci(pos, vel, mode)
        dep_deg = np.degrees(np.arcsin(np.dot(los, nadir)))
        assert 10 < dep_deg < 22, \
            f"Depression {dep_deg:.2f}° outside [10°, 22°]; expect ~15.73°"
```

### T5 — Tangent point altitude accuracy (two-level)
```python
def test_tangent_point_altitude():
    """
    Two-level test:
    (a) Physical requirement: tp_alt within ±5 km of 250 km (V1 criterion).
    (b) Implementation quality: tp_alt within 1 km — verifies that the
        iterative _compute_depression_angle is active, not the old single-
        estimate formula (which gave ~6 km error at polar latitudes).

    Test at multiple spacecraft positions spanning a half-orbit to sample
    both equatorial and polar geometries.
    """
    from astropy.time import Time
    from src.geometry.nb01_orbit_propagator_2026_04_05 import propagate_orbit

    df = propagate_orbit('2027-01-01T00:00:00', duration_s=5700.0, dt_s=570.0)
    H_TARGET = 250.0

    for _, row in df.iterrows():
        pos   = np.array([row.pos_eci_x, row.pos_eci_y, row.pos_eci_z])
        vel   = np.array([row.vel_eci_x, row.vel_eci_y, row.vel_eci_z])
        epoch = Time(row.epoch)
        for mode in ['along_track', 'cross_track']:
            los, _ = compute_los_eci(pos, vel, mode)
            tp     = compute_tangent_point(pos, los, epoch, H_TARGET)
            err    = abs(tp['tp_alt_km'] - H_TARGET)
            # (a) Physical requirement
            assert err < 5.0, (
                f"tp_alt {tp['tp_alt_km']:.2f} km, error {err:.2f} km > 5 km "
                f"at sc_lat={row.lat_deg:.1f}°, mode={mode}"
            )
            # (b) Implementation quality — catches regression to single-estimate
            assert err < 1.0, (
                f"Iterative depression angle should achieve < 1 km error, "
                f"got {err:.3f} km at sc_lat={row.lat_deg:.1f}°, mode={mode}. "
                f"Check _compute_depression_angle() uses iterative algorithm."
            )
```

```python
def test_tangent_point_leads_spacecraft():
    """
    Along-track tangent point must be AHEAD of spacecraft, > 500 km forward.
    Uses 3-D ECI dot product onto velocity unit vector.
```

```python
def test_vsc_los_ratio():
    """Along-track |V_sc_LOS| must be > 10× cross-track."""
    from astropy.time import Time
    from src.windmap.nb00_wind_map_2026_04_05 import UniformWindMap
    pos = np.array([0., 6.896e6, 0.])
    vel = np.array([977.2, 0., 7534.5])
    epoch = Time('2027-01-01T00:00:00', scale='utc')
    wind_map = UniformWindMap(v_zonal_ms=0.0, v_merid_ms=0.0)
    results = {}
    for mode in ['along_track', 'cross_track']:
        los, _ = compute_los_eci(pos, vel, mode)
        tp  = compute_tangent_point(pos, los, epoch)
        res = compute_v_rel(wind_map,
                            tp['tp_lat_deg'], tp['tp_lon_deg'], tp['tp_eci'],
```

### Critical implementation note — iterative depression angle

The `_compute_depression_angle()` function **must** use the iterative
algorithm described in §4.1. A simple `arccos((R_E + h_target) / (R_E +
altitude))` with a fixed mean sphere radius will cause T5 part (b) to fail
and will cause INT01 V1 to fail for polar-latitude epochs. The tell-tale
symptom is tp_alt deviations of ~4–6 km at high spacecraft latitudes (the
97.44° SSO has epochs at up to ~97° geocentric latitude).

The iterative steps are:
1. Seed R_tp with `_wgs84_geocentric_radius(sc_geocentric_lat) + h_target`.
2. Compute δ = arccos(R_tp / R_sc).
3. Project an approximate boresight forward by δ to estimate TP ECI.
4. Compute TP geocentric latitude from the approximate TP position.
5. Recompute R_tp = `_wgs84_geocentric_radius(tp_geocentric_lat) + h_target`.
6. Repeat from step 2 for `n_iter` (default 5) total iterations.
